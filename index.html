1. API

Es un conjunto de reglas y protocolos que permite que diferentes programas de software se comuniquen, intercambien datos y se aprovechen funcionalidades.

2. Agile

Metodología de desarrollo de software basada en la colaboración, la flexibilidad y la entrega contínua de valor.

3. Algoritmo

Un algoritmo es una secuencia finita y ordenada de instrucciones que se utilizan para resolver un problema específico o para realizar una tarea determinada.

4. Análisis de Requisitos

Proceso de recopilación y evaluación de las necesidades del usuario para definir las características del software.

5. Automatización de Procesos

La automatización de procesos en programación consiste en usar software y tecnología para convertir tareas manuales y repetitivas en procesos ejecutados automáticamente, liberando a los humanos para tareas más estratégicas.

6. Angular

Framework desarrollado por Google para construir aplicaciones web dinámicas utilizando TypeScript.

7. Arquitectura de Software

Se refiere al diseño de alto nivel de un sistema, definiendo su estructura, los componentes, sus relaciones y las decisiones de diseño fundamentales para cumplir con los requisitos de calidad del software, como la escalabilidad, mantenibilidad y rendimiento.

8. AOP (Programación Orientada a Aspectos)

Paradigma que permite separar preocupaciones transversales del código principal.

9. Back-end

Es la parte del desarrollo que no es visible para el usuario (el lado del servidor) y que se encarga de la lógica de la aplicación, la gestión de bases de datos, la autenticación de usuarios y la comunicación entre el servidor y el frontend.

10. Base de Datos

Sistema que almacena y organiza información de forma estructurada para su fácil acceso y manipulación.

11. Big Data

Es el conjunto de tecnologías y procesos para almacenar, gestionar y analizar conjuntos de datos masivos, complejos y de alta velocidad que superan la capacidad de las bases de datos y herramientas tradicionales.

12. Blockchain

Tecnología descentralizada que registra transacciones de forma segura e inmutable mediante un libro mayor digital.

13. Bug

Un bug en programación es un error o defecto en el código de un software que provoca un funcionamiento inesperado, incorrecto o no deseado.

14. Bootstrap

Bootstrap es un framework gratuito y de código abierto que utiliza HTML, CSS y JavaScript para ayudar a los desarrolladores web a crear sitios y aplicaciones web con diseño responsive.

15. Business Intelligence (BI)

es un conjunto de procesos, herramientas y metodologías que utilizan los desarrolladores y las empresas para recopilar, analizar y presentar datos empresariales de forma que se conviertan en información estratégica que apoye la toma de decisiones.

16. Bug Tracking

Sistema o proceso para registrar, rastrear y gestionar los errores detectados durante el desarrollo del software.

17. Cache

La caché es una memoria de alta velocidad que almacena datos de acceso frecuente para acelerar el acceso a ellos, reduciendo así la necesidad de consultar la memoria principal o el disco duro, que son más lentos.

18. CI/CD

CI/CD (Integración Continua y Despliegue Continuo) son prácticas de desarrollo de software que automatizan la creación, prueba y despliegue de código para acelerar los ciclos de lanzamiento y ofrecer versiones de forma más confiable.

19. Código Abierto (Open Source)

El Código Abierto (Open Source) es software cuyo código fuente está disponible públicamente, permitiendo a cualquiera inspeccionar, modificar y distribuir el programa libremente.

20. Compilador

Un compilador es una herramienta de software que traduce el código fuente de un lenguaje de programación, escrito en un lenguaje de alto nivel (legible por humanos), a código de bajo nivel o lenguaje máquina (legible por computadoras) para que el programa pueda ejecutarse.

21. CRUD (Create, Read, Update, Delete)

En programación, CRUD es un acrónimo de Crear (Create), Leer (Read), Actualizar (Update) y Eliminar (Delete), que son las cuatro operaciones fundamentales para gestionar datos persistentes en una aplicación o base de datos.

22. CSS (Cascading Style Sheets)

Lenguaje que define el estilo y diseño de las páginas web.

23. CORS (Cross-Origin Resource Sharing)

Mecanismo que permite a los servidores gestionar solicitudes de recursos desde distintos orígenes.

24. CLI (Command Line Interface)

CLI son las siglas de Command Line Interface o Interfaz de Línea de Comandos, un método de interacción con el sistema operativo o un programa a través de la escritura de comandos de texto en un terminal, en contraste con las interfaces gráficas que usan elementos visuales.

25. DevOps

DevOps es una cultura, un conjunto de prácticas y herramientas que combina el desarrollo (Dev) y las operaciones (Ops) de software para entregar aplicaciones y servicios de manera rápida, fiable y de alta calidad.

26. Docker

Docker es una plataforma de software de código abierto que permite a los desarrolladores crear, implementar y gestionar aplicaciones en contenedores.

27. Documentación

La documentación en programación es el conjunto de materiales textuales y visuales que describen y explican un software, su código, funcionamiento y uso, con el fin de hacerlo comprensible, mantenible y utilizable por desarrolladores, usuarios y otros interesados.

28. DTO (Data Transfer Object)

En programación, DTO significa Data Transfer Object (Objeto de Transferencia de Datos), y es un objeto simple diseñado para transportar datos entre procesos o capas de una aplicación, como entre el servidor y el cliente.

29. Debugging

Es el proceso de identificar, aislar y corregir errores (o "bugs") en el código de un programa o sistema para asegurar su funcionamiento correcto y eficiente.

30. Django

Framework de desarrollo web en Python que sigue el patrón MVC y agiliza la creación de aplicaciones robustas.

31. Despliegue

El despliegue en programación (también conocido como deployment) es el proceso de poner una aplicación o software en un entorno donde los usuarios finales pueden acceder y usarlo, es decir, pasarla del entorno de desarrollo al entorno de producción.

32. Design Patterns (Patrones de Diseño)

Soluciones reutilizables y probadas para problemas comunes en la ingeniería de software.

33. ERP (Enterprise Resource Planning)

Software de gestión que integra procesos empresariales en un único sistema.

34. Escalabilidad

Capacidad de un sistema para adaptarse y manejar un aumento en la carga de trabajo sin perder rendimiento.

35. ETL (Extract, Transform, Load)

Proceso de extracción, transformación y carga de datos desde diversas fuentes a un sistema de almacenamiento.

36. Endpoint

Punto de acceso para consumir servicios o datos de una API.

37. Encriptación

La encriptación en programación es el proceso de transformar información legible ("texto plano") en un formato ilegible ("texto cifrado") utilizando algoritmos matemáticos y una clave secreta, de modo que solo las personas autorizadas con la clave correcta puedan revertir el proceso y acceder a la información original.

38. E-Commerce

Comercio electrónico que se lleva a cabo a través de plataformas digitales.

39. E2E Testing (Testing de Extremo a Extremo)

Pruebas que evalúan el flujo completo de una aplicación, desde el inicio hasta su final.

40. Elasticsearch

Motor de búsqueda y análisis distribuido, ideal para manejar grandes volúmenes de datos en tiempo real.

41. Framework

Conjunto de herramientas y bibliotecas que facilitan el desarrollo estructurado de software.

42. Front-end

En programación, el front end se refiere a la parte de una aplicación web o sitio web con la que el usuario interactúa directamente; es todo lo visual, como el diseño, los botones, los menús y el contenido que ven. Esta parte se desarrolla utilizando tecnologías como HTML para la estructura, CSS para los estilos y JavaScript para la interactividad.

43. Función

Bloque de código reutilizable que realiza una tarea específica dentro de una aplicación.

44. Factorización de Código

La refactorización de código es la práctica de reestructurar el código fuente de un programa para mejorar su diseño interno sin cambiar su comportamiento externo ni su funcionalidad.

45. Firewall

Sistema de seguridad encargado de controlar el flujo de información en una red y prevenir accesos no autorizados.

46. Firebase

Plataforma de Google para el desarrollo de aplicaciones móviles y web con servicios en la nube integrados.

47. Figma

Herramienta colaborativa de diseño de interfaces y prototipos para aplicaciones.

48. FaaS (Function as a Service)

Modelo de computación en la nube donde se ejecutan funciones específicas de manera efímera y escalable.

49. Gestión de Versiones

La Gestión de Versiones en programación es el proceso de rastrear y controlar todos los cambios realizados en un proyecto de software, incluyendo el código fuente, documentos y otros artefactos, utilizando herramientas específicas.

50. Git

Sistema de control de versiones distribuido ampliamente utilizado para gestionar proyectos de desarrollo.

51. GraphQL

GraphQL es un lenguaje de consulta de API de código abierto y un entorno de ejecución del lado del servidor que permite a los clientes especificar de forma eficiente los datos que necesitan, recibiendo solo esa información y nada más.

52. Granularidad

Nivel de detalle en el que se dividen las funcionalidades o componentes de un sistema.

53. Gantt (Diagrama de Gantt)

Herramienta visual para planificar y gestionar proyectos a través de un cronograma.

54. Go (Golang)

Go (o Golang) es un lenguaje de programación de código abierto, compilado y de tipado estático, desarrollado por Google para ser simple, eficiente y fiable, especialmente útil para sistemas distribuidos, aplicaciones web de alta escala, servicios en la nube y para potenciar la concurrencia.

55. Grafana

Grafana es una plataforma de código abierto para visualización y análisis de datos que permite monitorear el rendimiento de aplicaciones e infraestructuras.

56. Git Flow

Git Flow es un modelo de ramificación para Git que proporciona una estructura para el desarrollo de software mediante ramas predefinidas como main (para código de producción), develop (para la integración de nuevas funcionalidades), y ramas temporales para features (características), releases (lanzamientos) y hotfixes (correcciones urgentes).

57. HTTP (Hypertext Transfer Protocol)

En informática, el HTTP (Protocolo de Transferencia de Hipertexto) es un protocolo de comunicación que define cómo los navegadores web (clientes) envían solicitudes a los servidores web para obtener y mostrar contenido, y cómo los servidores responden a esas solicitudes, posibilitando el funcionamiento de la World Wide Web.

58. Hashing

En programación, un hash es una cadena de caracteres alfanuméricos de longitud fija que se genera a partir de un algoritmo matemático aplicado a un dato de entrada, creando así una "huella digital" o "firma" única de ese dato.

59. Hosting

el hosting (alojamiento web) es un servicio que te proporciona un espacio en un servidor para almacenar todos los archivos de tu página web o aplicación (como textos, imágenes, bases de datos, etc.), haciéndolos accesibles a través de Internet.

60. Hypervisor

Un hipervisor es un software que permite ejecutar múltiples sistemas operativos (llamados máquinas virtuales) en una sola máquina física, gestionando y asignando de forma eficiente los recursos de hardware subyacentes (como CPU y memoria).

61. Heurística

Técnica de resolución de problemas basada en la experiencia y aproximaciones prácticas.

62. Hadoop

Framework de código abierto utilizado para el procesamiento distribuido de grandes volúmenes de datos.

63. HTML (HyperText Markup Language)

HTML es el Lenguaje de Marcado de Hipertexto (HyperText Markup Language), que es el lenguaje fundamental y estándar para crear y estructurar el contenido de una página web.

64. HATEOAS

Principio de diseño REST que introduce hipermedios como parte de la comunicación entre cliente y servidor.

65. IDE (Integrated Development Environment)

IDE (Entorno de Desarrollo Integrado) es una aplicación de software que combina en una única interfaz las herramientas necesarias para desarrollar código.

66. IA (Inteligencia Artificial)

La IA se refiere al desarrollo de sistemas informáticos que pueden realizar tareas que normalmente requieren inteligencia humana, como el aprendizaje, el razonamiento y la percepción.

67. Índice de Base de Datos

Estructura que mejora la velocidad de las consultas en una base de datos.

68. Inyección SQL

Técnica de ataque que explota vulnerabilidades en consultas SQL para acceder a datos no autorizados.

69. Interfaces de Usuario (UI)

70. IoT (Internet of Things)

Red de dispositivos conectados que recopilan y comparten datos a través de internet.

71. Immutable Infrastructure

Enfoque en el que los componentes del sistema no se modifican tras su despliegue; en su lugar, se reponen por completo.

72. IaaS (Infrastructure as a Service)

Modelo de servicio en la nube que entrega recursos informáticos virtualizados bajo demanda.

73. JSON

JSON (JavaScript Object Notation) es un formato de texto ligero y legible por humanos y máquinas para intercambiar datos.

74. Jenkins

Jenkins es un servidor de automatización de código abierto escrito en Java, usado principalmente para implementar las prácticas de Integración Continua (CI) y Entrega/Despliegue Continuo (CD) en el desarrollo de software.

75. Java

Lenguaje de programación orientado a objetos muy utilizado en aplicaciones empresariales.

76. JUnit

JUnit es un framework de código abierto para el lenguaje de programación Java que permite a los desarrolladores escribir y ejecutar pruebas unitarias automatizadas para verificar que las partes individuales de su código (como métodos o clases) funcionan correctamente

79. JUCE

80. Jest

Framework de pruebas para JavaScript, ampliamente usado para validar el comportamiento de aplicaciones y librerías.

81. Kubernetes

82. Kanban

es una metodología ágil para el desarrollo de software que utiliza un tablero visual para gestionar el flujo de trabajo y optimizar la eficiencia, identificando cuellos de botella y fomentando la colaboración.

83. Kotlin

Lenguaje de programación moderno que se ejecuta sobre la JVM y ofrece sintaxis concisa y seguridad en tipos.

84. Kafka

La "programación Kafka" se refiere al uso de Apache Kafka, una plataforma de transmisión de eventos de código abierto, para desarrollar aplicaciones que manejan flujos de datos en tiempo real.

86. Knockout.js

Knockout.js es una biblioteca JavaScript que facilita la creación de interfaces de usuario dinámicas y responsivas, aplicando el patrón de diseño Model-View-ViewModel (MVVM) para la sincronización automática entre los datos de la aplicación

87. Kibana

Herramienta de visualización que se integra con Elasticsearch para analizar y explorar datos.

88. Key-Value Store

Tipo de sistema de almacenamiento en el que los datos se gestionan en pares clave-valor.

89. Linux

Sistema operativo de código abierto ampliamente utilizado en servidores y entornos de desarrollo.

Solución (hardware o software) que distribuye la carga de trabajo entre varios servidores para optimizar el rendimiento.

91. Laravel

Framework de PHP que facilita el desarrollo de aplicaciones web modernas con una sintaxis elegante.

92. Linter

Herramienta que analiza el código fuente para detectar errores, malas prácticas y asegurar consistencia.

93. LDAP (Lightweight Directory Access Protocol)

94. Lua

95. LAMP (Linux, Apache, MySQL, PHP)

96. Lean

97. Microservicios

99. Maven

Herramienta de gestión y automatización de proyectos para el lenguaje Java.

100. MongoDB

Base de datos NoSQL orientada a documentos, reconocida por su alta escalabilidad y flexibilidad.

Patrón de diseño que separa la aplicación en tres componentes para facilitar el mantenimiento.

102. Machine Learning (Aprendizaje Automático)

Node.js es un entorno de ejecución de código abierto y multiplataforma que permite ejecutar JavaScript en el lado del servidor.

NoSQL en programación se refiere a un conjunto de bases de datos no relacionales que almacenan datos en formatos flexibles y variados.

116. OpenStack

Plataforma de software de código abierto para la creación y gestión de nubes privadas y públicas.

121. Python

123. PHP

Lenguaje de programación del lado del servidor, ampliamente usado para el desarrollo web.

Scrum es un marco de trabajo ágil para equipos de desarrollo de software que ayuda a gestionar proyectos complejos y adaptativos mediante ciclos de trabajo cortos (llamados Sprints), autoorganización, y entregas incrementales de valor al cliente.

146. SaaS (Software as a Service)

161. UX (User Experience)

Framework progresivo de JavaScript para la creación de interfaces de usuario interactivas en aplicaciones de una sola página.

173. Visual Studio

174. Version Control (Control de Versiones)

Sistema que permite rastrear, gestionar y revertir cambios en el código fuente.

175. Velocity

Métrica en metodologías ágiles que mide la cantidad de trabajo completado en un sprint o iteración.

176. Variable

Elemento que almacena datos en programación y que puede modificarse durante la ejecución del código.

177. WebSocket

Protocolo que permite comunicaciones bidireccionales y persistentes entre cliente y servidor en tiempo real.

178. WooCommerce

Plugin de comercio electrónico de código abierto para WordPress, que permite la creación de tiendas online.

179. WordPress

Sistema de gestión de contenidos (CMS) ampliamente utilizado para crear y administrar sitios web y blogs.

181. Windows

En informática, Windows es un conjunto de sistemas operativos (SO) desarrollados por Microsoft que permite a los usuarios interactuar con hardware y software en computadoras, laptops y tabletas, ofreciendo una interfaz gráfica de usuario (GUI) basada en ventanas, menús y iconos para facilitar el uso de programas, la navegación web y la gestión de archivos. Su función principal es administrar los recursos de la computadora, asegurando que todo opere de manera eficiente y segura.

182. WebRTC

Conjunto de estándares que facilita la comunicación en tiempo real (voz, video y datos) directamente en el navegador.

183. Webpack

Empaquetador de módulos que compila recursos y dependencias para aplicaciones web modernas.

184. Wireframe

Esquema o boceto preliminar del diseño de una interfaz que ayuda a planificar la disposición de elementos.

185. XML (eXtensible Markup Language)

En programación, XML (Extensible Markup Language) es un lenguaje de marcado que sirve para almacenar y transmitir datos de forma estructurada, legible tanto por humanos como por máquinas.

Vulnerabilidad de seguridad que permite inyectar scripts maliciosos en páginas web vistas por otros usuarios.

187. Xcode

Entorno de desarrollo integrado (IDE) de Apple utilizado para crear aplicaciones para iOS, macOS, watchOS y tvOS.

En programación, XPath (XML Path Language) es un lenguaje de expresión que permite navegar y seleccionar nodos (elementos, atributos, etc.) dentro de un documento XML o HTML, similar a cómo se usan las expresiones regulares para encontrar texto en un archivo.

Lenguaje diseñado para consultar y transformar datos almacenados en documentos XML.

190. Xamarin

Xamarin es una plataforma de desarrollo de aplicaciones móviles multiplataforma que permite a los desarrolladores crear aplicaciones nativas para iOS, Android y Windows usando el lenguaje de programación C# y la plataforma .NET.

Variante de HTML que sigue las reglas de XML, asegurando una sintaxis más estricta.

192. Xterm

Emulador de terminal para sistemas Unix, utilizado para interactuar con la línea de comandos.

193. YAML (YAML Ain’t Markup Language)

Formato legible para humanos empleado comúnmente en archivos de configuración y manejo de datos.

194. YARN (Yet Another Resource Negotiator)

En programación, YARN puede referirse a dos conceptos distintos: YARN (Yet Another Resource Negotiator) en el ecosistema de Hadoop, un sistema para gestionar recursos de clúster y ejecutar aplicaciones de procesamiento de datos.

195. Yii

Yii es un framework PHP de alto rendimiento basado en componentes, diseñado para facilitar el desarrollo rápido de aplicaciones web modernas y a gran escala.

196. Yeoman

Herramienta de scaffolding que ayuda a generar la estructura inicial de nuevos proyectos web.

197. YUI (Yahoo! User Interface library)

Biblioteca de JavaScript y recursos CSS desarrollada por Yahoo! para facilitar la creación de interfaces web.

198. YACC (Yet Another Compiler Compiler)

Herramienta utilizada para generar analizadores sintácticos en el desarrollo de compiladores.

199. Yottabyte

Unidad de medida de información que equivale a 10^24 bytes, usada para cuantificar grandes volúmenes de datos.

200. Yield

Palabra clave en algunos lenguajes de programación que permite retornar valores de funciones generadoras de forma paulatina.

201. Z-Score

En programación, el Z-Score es un valor estadístico que mide cuántas desviaciones estándar está un punto de datos alejado de la media de un conjunto de datos, permitiendo estandarizar y comparar datos de diferentes escalas.

202. Zero-Day

Vulnerabilidad de seguridad desconocida por los desarrolladores, la cual puede ser explotada antes de su corrección.

203. Zend

Framework y conjunto de herramientas para el desarrollo de aplicaciones web en PHP, proporcionado por Zend Technologies.

204. ZShell (Zsh)

Intérprete de comandos para sistemas Unix conocido por su alta configurabilidad y características avanzadas.

205. Zookeeper

Servicio centralizado que facilita la coordinación y configuración en sistemas distribuidos, fundamental en arquitecturas Big Data.

206. Zig

Lenguaje de programación orientado a la eficiencia y seguridad, considerado como una alternativa moderna a C.

207. Zigbee

Protocolo de comunicación inalámbrica de bajo consumo y baja velocidad, utilizado en redes de sensores y domótica.

208. Zonal Architecture

Enfoque de diseño de sistemas que segmenta funcionalidades en zonas o módulos para aislar y gestionar la complejidad.

209. Codex

Es un modelo de OpenAI para la generación de código. Se basa en la arquitectura GPT y está entrenado con una gran cantidad de código público. Puede traducir lenguaje natural a código.

210. Whisper

Un modelo de OpenAI para el reconocimiento automático de voz. Es de código abierto y multilingüe, entrenado con una vasta diversidad de audio.

211. MidJourney

Una herramienta de IA para la creación de arte visual. Utiliza redes neuronales para generar imágenes a partir de descripciones de texto (prompts).

212. Chatbot

Un programa de IA que interactúa con usuarios mediante lenguaje natural. Estos programas pueden ser desde sistemas simples basados en reglas hasta modelos complejos que usan el procesamiento de lenguaje natural (NLP).

213. Asistente virtual

Es un sistema de IA diseñado para ayudar en tareas cotidianas. A diferencia de un chatbot, un asistente virtual como Siri o Alexa a menudo está integrado en un sistema operativo y puede ejecutar comandos para programar citas, enviar mensajes o controlar otros dispositivos.

214. Agente autónomo

Un sistema de IA capaz de operar sin supervisión humana. Estos agentes tienen objetivos a largo plazo y pueden tomar decisiones para lograrlos. Un ejemplo es un vehículo autónomo, que decide rutas y acciones sin intervención constante.

215. Sistemas expertos

Programas que imitan la toma de decisiones humanas en un dominio específico. Usan una base de conocimiento (datos y reglas) y un motor de inferencia para resolver problemas complejos.

216. IA simbólica

El primer enfoque de IA, basado en reglas y lógica formal. Se centra en representar el conocimiento y el razonamiento humano a través de símbolos y reglas lógicas.

217. IA conexionista

Este enfoque se basa en redes neuronales artificiales, inspiradas en la estructura del cerebro.

218. IA fuerte

Es una IA hipotética con capacidades de razonamiento general similares a las humanas. No solo simula la inteligencia, sino que la posee realmente, incluyendo conciencia, subjetividad y la capacidad de entender y aplicar su conocimiento.

219. IA débil

Son los sistemas de IA que existen hoy en día, diseñados para tareas específicas sin conciencia propia. Son muy eficientes en una tarea concreta, como jugar al ajedrez o reconocer imágenes.

220. Ética en IA

Es la disciplina que analiza el impacto social y moral de la inteligencia artificial. Se enfoca en cuestiones como el sesgo algorítmico, la privacidad de los datos y la responsabilidad de los errores.

221. Sesgo algorítmico

Una distorsión en los resultados de la IA que se debe a datos o un diseño defectuoso. Esto ocurre cuando un algoritmo refleja los sesgos presentes en los datos de entrenamiento.

222. Caja negra en IA

Es una situación en la que no se puede entender cómo un modelo toma sus decisiones. Esto es común en modelos complejos, como las redes neuronales profundas, lo que genera problemas de confianza.

223. Interpretabilidad

La capacidad de explicar cómo un modelo de IA produce sus resultados. A diferencia de la explicabilidad, la interpretabilidad se refiere a la facilidad general de comprender las relaciones entre las entradas y las salidas del modelo.

224. Explicabilidad (XAI)

Un campo que busca hacer transparentes los modelos de IA complejos. La XAI (eXplainable Artificial Intelligence) desarrolla herramientas para entender las decisiones de los modelos de "caja negra", lo cual es crucial en sectores como la medicina.

225. Data labeling

El proceso de etiquetar datos para entrenar modelos supervisados. Es la tarea de asignar categorías a los datos (por ejemplo, "perro" a una foto) para que el modelo aprenda a hacer predicciones.

226. Crowdsourcing de datos

La recolección de datos etiquetados mediante la colaboración masiva. Se usa para recopilar grandes volúmenes de datos de manera eficiente.

227. Data mining

El proceso de descubrir patrones útiles en grandes volúmenes de datos. Es la etapa inicial del análisis de datos, usando métodos estadísticos para identificar tendencias y correlaciones.

228. Feature engineering

La selección y creación de variables relevantes para un modelo. Consiste en transformar los datos crudos en variables que el modelo pueda utilizar para mejorar su rendimiento.

229. Pipeline de IA

La secuencia de pasos desde la recolección de datos hasta la inferencia. Es un flujo de trabajo automatizado que incluye la limpieza de datos, el entrenamiento, la validación y el despliegue del modelo.

230. Inferencia

El uso de un modelo ya entrenado para hacer predicciones. Es la fase de producción de la IA, donde el modelo recibe nuevos datos y genera un resultado.

231. Latencia en IA

El tiempo que tarda un modelo en dar una respuesta. Es un factor crítico en aplicaciones en tiempo real, como la conducción autónoma o la detección de fraude, donde las decisiones deben ser instantáneas.

232. Edge AI

La implementación de inteligencia artificial en dispositivos locales, sin necesidad de la nube. Permite procesar datos directamente en el dispositivo (por ejemplo, en un smartphone) para reducir la latencia y mejorar la privacidad.

233. Federated Learning

Una técnica para entrenar modelos de IA sin compartir datos entre dispositivos. Solo se comparten las actualizaciones del modelo, lo que protege la información personal.

234. Model compression

Un conjunto de técnicas para reducir el tamaño de los modelos sin perder precisión. Es esencial para desplegar modelos en dispositivos con recursos limitados.

235. Quantization

El proceso de reducir la precisión de los cálculos de un modelo para hacerlo más eficiente. Esto disminuye el tamaño del modelo y acelera la inferencia.

236. Pruning

La eliminación de neuronas o conexiones innecesarias en una red. Es como podar un árbol: se eliminan las partes que no contribuyen significativamente al rendimiento del modelo para hacerlo más ligero.

237. Knowledge distillation

La transferencia del conocimiento de un modelo grande a uno más pequeño. Un modelo "maestro" entrena a un modelo "estudiante" para que imite su comportamiento y así lograr una alta precisión con un modelo más eficiente.

238. Benchmark en IA

Un conjunto de pruebas estandarizadas que se usa para medir el rendimiento de los modelos y comparar su desempeño de manera justa.

239. GPT

Una familia de modelos de lenguaje, como ChatGPT, que pueden generar texto coherente y fluido.

240. BERT

Un modelo de lenguaje de Google basado en Transformers, conocido por entender el contexto de las palabras en una oración.

241. Colab

Google Colab es un servicio gratuito en la nube, basado en cuadernos Jupyter, que facilita la escritura y ejecución de código Python para tareas de aprendizaje automático, ciencia de datos y educación, sin necesidad de instalación previa.

242. T5 (Text-to-Text Transfer Transformer)

Un modelo de Google que formula cualquier tarea de lenguaje (traducción, resumen) como una conversión de texto a texto.

243. Stable Diffusion

Un modelo de IA generativa que crea imágenes realistas a partir de descripciones de texto.

244. DALL·E

Un modelo de OpenAI que genera imágenes y arte a partir de descripciones de texto.

245. Imagen AI

Un modelo de Google especializado en la generación de imágenes de alta calidad.

246. Reinforcement Learning Agent

Un agente de IA que aprende a tomar decisiones a través de la prueba y error en un entorno.

247. Recompensa

La señal positiva que guía al agente en el aprendizaje por refuerzo.

248. Castigo

La señal negativa que corrige el comportamiento del agente.

249. Política (policy)

La estrategia que sigue un agente para decidir qué acción tomar.

250. Entorno simulado

Un espacio virtual usado para entrenar agentes de IA de manera segura y controlada.

251. AlphaGo

Una IA de Google DeepMind que venció a campeones humanos en el juego de Go. Es un ejemplo famoso de aprendizaje por refuerzo.

252. Transformers

Una arquitectura de red neuronal clave para el procesamiento del lenguaje y otros datos secuenciales.

253. Atención (attention)

Un mecanismo dentro de los modelos que les permite enfocarse en las partes más importantes de la información.

254. Encoder

La parte de un Transformer que procesa los datos de entrada.

255. Decoder

La parte de un Transformer que genera la salida, como texto o imágenes.

256. Autoencoder

Una red neuronal que aprende a comprimir datos y luego a reconstruirlos.

257. GAN (Generative Adversarial Network)

Un tipo de red neuronal que enfrenta a un modelo generador (que crea datos) y un modelo discriminador (que evalúa su autenticidad).

258. Modelo generativo

Un sistema de IA que crea nuevos datos, como texto o imágenes, a partir de patrones aprendidos.

259. Modelo discriminativo

Un modelo de IA que clasifica datos existentes en categorías.

260. Tokenización

Dividir texto en unidades más pequeñas, como palabras o frases, para que un modelo las procese.

261. Embeddings

Representaciones numéricas que capturan el significado de palabras o datos.

262. Word2Vec

Un modelo que crea representaciones numéricas (vectores) de palabras.

263. GloVe

Un algoritmo de embeddings que utiliza estadísticas globales del texto para crear representaciones de palabras.

264. Transfer Learning

Usar un modelo ya entrenado para una tarea similar. Es como reutilizar el conocimiento.

265. Fine-tuning

Ajustar un modelo preentrenado con un nuevo conjunto de datos para una tarea específica. Es el proceso de "especializar" el modelo.

266. Air gap

Una estrategia de seguridad que aísla físicamente un sistema crítico de internet para protegerlo de ciberataques.

267. Hardening

El proceso de reforzar la seguridad de un sistema al eliminar vulnerabilidades.

268. Parches de seguridad

Actualizaciones de software que corrigen fallos y vulnerabilidades.

269. Modal

Una ventana emergente que se superpone a la interfaz principal y requiere que el usuario interactúe con ella para poder continuar.

270. Text box

Un elemento de interfaz que combina un campo de texto con una lista desplegable de opciones.

302. HTTPS (HyperText Transfer Protocol Secure)

303. URL (Uniform Resource Locator)

304. SSL/TLS

305. CMS (Content Management System)

306. SEO (Search Engine Optimization)

Otra plataforma de gestión de repositorios Git.

La plataforma de servicios en la nube de Microsoft.

321. Google Cloud Platform (GCP)

Una red neuronal diseñada para aprender una representación comprimida de los datos (codificación) y luego reconstruirlos a partir de esa representación (decodificación).

325. Variational Autoencoder (VAE)

Un tipo de autoencoder que, en lugar de mapear a un único punto, mapea a una distribución, lo que le permite generar nuevos datos que se parecen al conjunto de entrenamiento.

326. Feature extraction (Extracción de Características)

El proceso de identificar y seleccionar las características más relevantes de los datos sin modificar su naturaleza, lo cual es vital para reducir la complejidad.

327. Dimensionality reduction (Reducción de Dimensionalidad)

La técnica de disminuir el número de variables en un dataset mientras se conserva la mayor cantidad de información clave posible.

328. PCA (Principal Component Analysis)

Una técnica de reducción de dimensionalidad que transforma las variables originales en un nuevo conjunto de variables no correlacionadas llamadas componentes principales.

329. t-SNE

Un algoritmo no lineal que se usa principalmente para visualizar datos de alta dimensión en un espacio 2D o 3D.

330. UMAP (Uniform Manifold Approximation and Projection)

Un método de reducción de dimensionalidad que es más rápido y escalable que t-SNE, manteniendo una estructura topológica similar de los datos.

331. Clustering

Una tarea de aprendizaje no supervisado que agrupa puntos de datos sin etiquetas según sus similitudes.

332. K-means

Un algoritmo de clustering que agrupa los datos alrededor de un número predefinido (K) de centroides.

333. DBSCAN

Un algoritmo de clustering que agrupa puntos en función de la densidad de los datos.

334. Hierarchical Clustering

Un método de agrupación que construye una estructura de árbol (dendrograma) para visualizar las relaciones jerárquicas entre los grupos.

335. Classification (Clasificación)

La tarea de IA que asigna categorías discretas (ej. "perro" o "gato") a las entradas de datos.

336. Regression (Regresión)

La tarea de IA que predice valores numéricos continuos (ej. precio de una casa o temperatura).

337. Decision Tree (Árbol de Decisión)

Un modelo intuitivo que usa una estructura de árbol para tomar decisiones basadas en divisiones jerárquicas de las características de los datos.

338. Random Forest

Un método de Ensemble Learning que construye múltiples árboles de decisión y combina sus predicciones para mejorar la precisión y evitar el sobreajuste.

339. Gradient Boosting

Una técnica de Ensemble Learning que entrena una secuencia de modelos débiles (usualmente árboles) donde cada nuevo modelo corrige los errores del anterior.

340. XGBoost (eXtreme Gradient Boosting)

Una implementación optimizada y altamente eficiente del gradient boosting, muy usada para ganar competencias de ciencia de datos.

341. LightGBM

Un algoritmo de gradient boosting conocido por su alta velocidad y eficiencia en el manejo de grandes datasets.

342. CatBoost

Un algoritmo de boosting desarrollado por Yandex que está especializado en manejar automáticamente variables categóricas.

343. Ensemble Learning

La práctica de usar múltiples modelos combinados para mejorar la robustez y el rendimiento predictivo general.

344. Bagging (Bootstrap Aggregating)

Una técnica de ensemble que entrena modelos en subconjuntos aleatorios de datos con reemplazo.

345. Stacking

Un método de ensemble que utiliza un modelo meta para combinar las predicciones de varios modelos base.

346. Blending

Una variante de stacking donde la combinación se hace con una división de datos diferente para evitar fugas de información.

347. Feature selection (Selección de Características)

El proceso de elegir el subconjunto de variables más relevantes para el modelo.

348. Hyperparameter tuning (Optimización de Hiperparámetros)

El proceso de ajustar los parámetros que definen la estructura del modelo para mejorar su rendimiento.

349. Grid Search

Un método exhaustivo que prueba sistemáticamente todas las combinaciones de hiperparámetros definidas.

350. Random Search

Un método aleatorio que selecciona combinaciones de hiperparámetros de forma aleatoria, a menudo más eficiente que Grid Search.

351. Bayesian Optimization

Una estrategia probabilística que utiliza resultados anteriores para guiar la búsqueda del siguiente mejor conjunto de hiperparámetros.

352. Reinforcement Signal (Señal de Refuerzo)

La recompensa (positiva) o el castigo (negativo) que el agente de RL recibe de su entorno.

353. Policy (Política)

La estrategia o conjunto de reglas que el agente sigue para decidir qué acción tomar en un estado dado.

354. Value Function (Función de Valor)

Una función que estima qué tan bueno es un estado o una acción a largo plazo para el agente de RL.

355. Q-Learning

Un algoritmo de RL que aprende el valor óptimo de acción-estado (función Q) sin necesidad de un modelo del entorno.

356. SARSA (State-Action-Reward-State-Action)

Un algoritmo de RL similar a Q-learning, pero que aprende de la política actualmente seguida por el agente.

357. Actor-Critic

Un modelo de RL que utiliza dos redes neuronales: el **Actor** define la política y el **Critic** estima la función de valor.

358. Markov Decision Process (MDP)

El marco matemático subyacente que formaliza los problemas de toma de decisiones secuenciales en entornos estocásticos (probabilísticos).

359. Cloud Computing

El uso de recursos de computación (servidores, almacenamiento, aplicaciones) a través de Internet bajo demanda.

360. Cloud bursting

Una técnica que combina recursos locales (privados) con la nube pública para manejar picos repentinos de demanda.

361. Multi-cloud

El uso estratégico de servicios de múltiples proveedores de nube para evitar la dependencia de uno solo.

362. Hybrid cloud

La combinación de una nube privada (propia) con una nube pública (proveedor).

363. Private cloud

Una infraestructura en la nube dedicada exclusivamente a una sola organización.

364. Public cloud

Una infraestructura compartida por múltiples usuarios y gestionada por un proveedor externo (AWS, Azure, GCP).

365. Cloud security

El conjunto de políticas, tecnologías y controles para proteger los datos, aplicaciones e infraestructura en la nube.

366. Cloud migration

El proceso de mover los activos de tecnología de la información (aplicaciones, datos) de un entorno local a la nube.

367. DevOps

Una cultura y práctica que busca integrar los equipos de Desarrollo (Dev) y Operaciones (Ops) para automatizar y mejorar el ciclo de vida de la entrega de software.

368. DevSecOps

Una extensión de DevOps que integra la **Seguridad (Sec)** en cada etapa del ciclo de vida del desarrollo.

369. Pipeline

Un flujo de trabajo automatizado que gestiona la construcción, prueba y despliegue del software (CI/CD).

370. Container Orchestration

La gestión automatizada de contenedores (como Docker) en entornos de producción, encargándose de la escala, la salud y el networking.

371. Canary Deployment

Una estrategia de despliegue gradual donde la nueva versión se lanza a un pequeño subconjunto de usuarios para probarla en producción antes de un despliegue completo, minimizando riesgos.

372. Blue-Green Deployment

Una técnica de deployment que usa dos entornos idénticos (**Blue** - activo, **Green** - reserva). Se despliega la nueva versión en el entorno inactivo (Green) y, una vez probada, el tráfico se conmuta instantáneamente, reduciendo el tiempo de inactividad.

373. Rolling Deployment

La actualización progresiva de la nueva versión, reemplazando servidores uno por uno en lugar de todos a la vez.

374. Infrastructure as Code (IaC)

La gestión y aprovisionamiento de infraestructura mediante archivos de configuración o scripts, en lugar de procesos manuales.

375. Terraform

Una herramienta de IaC que permite definir y aprovisionar infraestructura en la nube (AWS, Azure, etc.) mediante un lenguaje declarativo.

376. Ansible

Una herramienta de automatización que simplifica la configuración y el despliegue en múltiples máquinas.

377. Puppet & Chef

Plataformas de gestión de configuración que ayudan a mantener un estado deseado en los servidores.

378. Jenkins

Un servidor de automatización de código abierto muy popular para implementar Pipelines de CI/CD.

379. Continuous Monitoring

La vigilancia constante y automática de la salud y el rendimiento de los sistemas en producción.

380. Observability

La capacidad de medir el estado interno de un sistema complejo a partir de los datos que emite (logs, métricas y traces).

381. Repository

El almacén donde se guarda el código fuente de un proyecto y todo su historial de cambios.

382. Commit

Un registro de cambios realizado en un momento específico en el historial del repositorio.

383. Branch

Una rama de desarrollo que permite trabajar en paralelo en nuevas características o correcciones sin afectar el código principal.

384. Merge

La acción de combinar los cambios de una branch a otra.

385. Pull Request

Una solicitud para que los cambios hechos en una branch sean revisados y fusionados en la rama principal.

386. Fork

Una copia de un repositorio que permite al usuario trabajar de forma independiente.

387. Clone

Una copia local de un repositorio remoto para trabajar en ella desde el ordenador personal.

388. Tag

Un marcador inmutable usado para señalar un punto importante en la historia del repositorio, como una versión final.

389. Release

Una versión estable y verificada del software lista para su distribución y uso.

390. CPU (Unidad Central de Procesamiento)

El **"cerebro"** del computador, encargado de ejecutar instrucciones y coordinar todas las operaciones.

391. Procesador multinúcleo

Una CPU que integra varios núcleos de procesamiento para manejar múltiples tareas en paralelo.

392. Caché de CPU

Una memoria ultrarrápida integrada en el procesador para almacenar datos de acceso frecuente y acelerar la ejecución.

393. GPU (Unidad de Procesamiento Gráfico)

Un procesador especializado en cálculos paralelos, ideal para gráficos, videojuegos e inteligencia artificial.

394. RAM (Memoria de Acceso Aleatorio)

Una memoria volátil y rápida donde se almacenan temporalmente los programas y datos en uso.

395. ROM (Memoria de Solo Lectura)

Una memoria no volátil que almacena permanentemente instrucciones esenciales del sistema (firmware).

396. Placa base (Motherboard)

La tarjeta de circuito principal que conecta y permite la comunicación entre todos los componentes de hardware.

397. Almacenamiento (Disco duro/SSD)

En el desarrollo de software, la elección del almacenamiento, principalmente entre un disco duro (HDD) y una unidad de estado sólido (SSD), es crucial para optimizar el rendimiento, la velocidad y la eficiencia del flujo de trabajo.

398. HDD (Disco duro)

Almacenamiento magnético tradicional, de gran capacidad, pero más lento.

399. SSD (Unidad de Estado Sólido)

Almacenamiento basado en memoria flash, mucho más rápido y eficiente que el HDD.

400. Energía

La energía en el desarrollo de software se puede entender desde dos perspectivas: el consumo energético que genera la actividad de desarrollo y las aplicaciones de software que gestionan y optimizan la energía.

401. Fuente de poder (PSU)

Convierte la electricidad de la toma de corriente a los voltajes requeridos por los componentes.

402. Tarjeta de red (NIC)

Permite la conexión a la red (cableada o inalámbrica).

403. Tarjeta de sonido

En el desarrollo de software, la tarjeta de sonido es la interfaz de hardware que permite a los programas interactuar con el audio de una computadora. Los desarrolladores escriben código que se comunica con los controladores de la tarjeta para controlar la entrada y salida de sonido.

404. Disipador térmico

Bloque de metal que transfiere el calor generado por componentes como la CPU al aire circundante.

405. Pasta térmica

Un compuesto que se aplica entre la CPU y el disipador para mejorar la transferencia de calor.

406. Ventilador de CPU

En el desarrollo de software, un ventilador de CPU es crucial para mantener el rendimiento y la estabilidad del equipo, especialmente durante tareas intensivas como la compilación de código, el renderizado de gráficos y la ejecución de máquinas virtuales. El sobrecalentamiento de la CPU puede ralentizar el sistema o provocar fallos inesperados.

407. Firmware de Arranque

El firmware de arranque es un software fundamental, almacenado en la memoria no volátil de un dispositivo, que se ejecuta inmediatamente después de encenderlo para inicializar el hardware y cargar el sistema operativo.

408. BIOS (Sistema Básico de Entrada/Salida)

El firmware tradicional que inicializa el hardware y comienza la carga del sistema operativo.

409. UEFI (Interfaz de Firmware Extensible Unificado)

La evolución del BIOS, que ofrece una interfaz más moderna y funciones de seguridad mejoradas.

410. Torre de PC (chasis)

La estructura o caja que alberga y protege todos los componentes del computador.

411. Ofimática

Acrónimo de "oficina" e "informática". Es el conjunto de herramientas, técnicas y aplicaciones informáticas que se usan para optimizar, automatizar y mejorar las tareas y procedimientos de oficina.

412. Suite ofimática

Conjunto de programas informáticos que se distribuyen juntos y que incluyen las herramientas más comunes para las tareas de oficina, como el procesador de textos, la hoja de cálculo y el software de presentaciones.

413. Productividad

Medida de la eficiencia de una persona o sistema al realizar tareas. La ofimática busca aumentar la productividad al agilizar los procesos.

414. Automatización

Proceso de usar la tecnología para realizar tareas de forma automática, sin la intervención manual del usuario. Es un objetivo clave de la ofimática.

415. Digitalización

Proceso de convertir información analógica (como documentos en papel) en formato digital, para que pueda ser gestionada y almacenada mediante herramientas ofimáticas.

416. Computación en la nube (Ofimática)

Almacenamiento y acceso a datos y programas a través de internet, en lugar de hacerlo en un disco duro local. Permite el trabajo colaborativo en las herramientas ofimáticas.

417. Software libre (Ofimática)

Programas informáticos que otorgan a los usuarios la libertad de usar, estudiar, cambiar y distribuir el software. Existen suites ofimáticas de software libre, como LibreOffice.

418. Software propietario (Ofimática)

Programas informáticos que son propiedad exclusiva de una empresa, que restringe su uso, copia y distribución.

419. Interfaz de usuario (UI) (Ofimática)

El medio a través del cual una persona interactúa con un programa o dispositivo. Las suites ofimáticas suelen tener interfaces de usuario comunes entre sus aplicaciones.

420. Compatibilidad (Ofimática)

La capacidad de un programa para funcionar con otros programas, formatos de archivo o hardware. Un aspecto crucial en el intercambio de documentos ofimáticos.

421. Procesador de textos

Aplicación de software utilizada para la creación, edición, formato, guardado y transmisión de documentos de texto. Ejemplos incluyen Microsoft Word y Google Docs.

422. Formato de texto

Conjunto de reglas que determinan la apariencia visual del texto, incluyendo el tipo de letra, tamaño, color y estilo (negrita, cursiva, subrayado).

423. Plantilla (Documento)

Documento preformateado que sirve como punto de partida para crear nuevos documentos, como cartas, currículos o informes, manteniendo un diseño consistente.

424. Encabezado y pie de página

Áreas en los márgenes superior e inferior de cada página de un documento, respectivamente, que pueden contener información como números de página, título o fecha.

425. Tabla de contenido

Listado automático de títulos y subtítulos de un documento, con los números de página correspondientes, que facilita la navegación.

426. Combinación de correspondencia

Herramienta para crear cartas, etiquetas o sobres personalizados para múltiples destinatarios, utilizando una plantilla y una fuente de datos (como una hoja de cálculo).

427. Corrector ortográfico y gramatical

Función que revisa y sugiere correcciones para errores de ortografía y gramática en un documento.

428. Hipervínculo

Enlace que conecta una parte del documento con otra ubicación, como una página web o un archivo diferente.

429. Hoja de cálculo

Aplicación de software que permite organizar, analizar y almacenar datos en formato de tabla, compuesta por filas y columnas. Ejemplos incluyen Microsoft Excel y Google Sheets.

430. Celda

La intersección de una fila y una columna en una hoja de cálculo. Es la unidad básica para almacenar datos.

431. Fórmula (Hoja de cálculo)

Expresión matemática que realiza cálculos con los valores de las celdas en una hoja de cálculo.

432. Función (Hoja de cálculo)

Fórmulas predefinidas que realizan cálculos específicos, como la suma (**SUMA**), el promedio (**PROMEDIO**) o el valor máximo (**MAX**).

433. Gráfico (Hoja de cálculo)

Representación visual de los datos de una hoja de cálculo, lo que facilita su análisis e interpretación.

434. Tabla dinámica

Herramienta que resume, agrupa y analiza grandes cantidades de datos, permitiendo una visión flexible de la información.

435. Macro

Serie de comandos grabados que se ejecutan automáticamente para automatizar tareas repetitivas en una hoja de cálculo.

436. Filtrar datos

Proceso de mostrar solo los datos que cumplen con ciertos criterios, ocultando el resto de la información de la hoja de cálculo.

437. Software de presentaciones

Aplicación para crear diapositivas que se utilizan para apoyar una exposición oral. Ejemplos incluyen Microsoft PowerPoint y Google Slides.

438. Diapositiva

Cada una de las páginas individuales que componen una presentación.

439. Animación (Presentación)

Efecto visual que se aplica a un objeto de la diapositiva, como texto o imágenes, para que aparezca o se mueva durante la presentación.

440. Transición

Efecto visual que se produce al cambiar de una diapositiva a otra.

441. Plantilla de diseño (Presentación)

Conjunto de fondos, combinaciones de colores, fuentes y otros elementos de diseño que se aplican a las diapositivas de una presentación para darles una apariencia uniforme.

442. Modo de vista de presentador

Vista que permite al orador ver sus notas y la próxima diapositiva en una pantalla, mientras la audiencia solo ve la diapositiva actual.

443. Correo electrónico

Servicio que permite el envío y recepción de mensajes electrónicos a través de una red. Las herramientas ofimáticas a menudo incluyen un gestor de correo, como Microsoft Outlook.

444. Agenda

Herramienta de software para organizar y gestionar citas, tareas y eventos. A menudo se integra con el correo electrónico y los calendarios.

445. Calendario

Función que permite visualizar y organizar eventos, citas y reuniones de manera cronológica.

446. Almacenamiento en la nube (Ofimática)

Servicio que permite guardar archivos en servidores remotos accesibles a través de internet, facilitando el acceso y la sincronización desde cualquier dispositivo.

447. Trabajo colaborativo

Posibilidad de que varias personas trabajen de forma simultánea en el mismo documento o proyecto a través de herramientas ofimáticas en línea.

448. Hardware ofimático

Componentes físicos necesarios para el funcionamiento de las herramientas ofimáticas, como ordenadores, impresoras y escáneres.

449. Impresora

Dispositivo periférico que produce una copia física (en papel) de un documento o imagen digital.

450. Escáner

Dispositivo que convierte documentos o imágenes físicas en archivos digitales.

451. Periférico

Cualquier dispositivo que se conecta al ordenador para añadir funcionalidades, como una impresora, un escáner o un teclado.

452. Copia de seguridad (Backup)

Proceso de crear duplicados de los datos para poder recuperarlos en caso de pérdida o daño.

453. Cifrado de datos

Técnica que codifica la información para protegerla de accesos no autorizados, garantizando su confidencialidad.

454. Carpeta

Directorio virtual que se utiliza para organizar archivos en el sistema de almacenamiento.

455. Archivo

Conjunto de datos almacenados en el ordenador, como un documento de texto, una hoja de cálculo o una presentación.

456. Formato de archivo

Estructura específica en la que se guardan los datos de un archivo, como **.docx**, **.xlsx** o **.pdf**.

457. PDF (Formato de Documento Portátil)

Formato de archivo universal que permite ver documentos con un diseño fijo en cualquier dispositivo, independientemente del software que se use.

458. Compresión de archivos

Proceso de reducir el tamaño de un archivo para que ocupe menos espacio de almacenamiento y sea más fácil de transmitir.

459. Emulador

Un software o hardware que permite a un sistema informático (conocido como **host**) imitar las funciones de otro sistema (llamado **invitado**).

460. Laptop (Portátil)

Un computador compacto con pantalla, teclado y batería integrados para la movilidad.

461. Ultrabook

Un tipo de portátil extremadamente delgado, liviano y enfocado en la eficiencia energética.

462. Tablet

Un dispositivo móvil que usa una pantalla táctil como interfaz principal.

463. Smartphone

Un teléfono avanzado que integra funciones de computador, con procesador, memoria y sensores.

464. Servidor

Una computadora potente dedicada a proveer servicios y recursos a otras máquinas en una red.

465. Supercomputadora

Un sistema con enorme poder de cálculo utilizado para investigación y simulaciones científicas a gran escala.

466. Mainframe

Computadoras de gran capacidad usadas por grandes organizaciones (bancos, gobiernos) para gestionar datos masivos y transacciones críticas.

467. Workstation

Una computadora de alto rendimiento optimizada para tareas técnicas, como diseño gráfico, ingeniería o edición de video.

468. Microcontrolador

Un chip pequeño que integra CPU, memoria y periféricos, diseñado para sistemas embebidos y tareas de control (ej. en electrodomésticos).

469. FPGA (Matriz de Puertas Programables en Campo)

Un circuito integrado reprogramable que se puede configurar para realizar tareas de hardware muy específicas y rápidas.

470. Raspberry Pi

Una computadora de placa única (**SBC**) de bajo costo y tamaño reducido, muy popular en proyectos educativos y de electrónica.

471. Arduino

Una placa de hardware libre diseñada para crear prototipos de electrónica de forma sencilla, ideal para interactuar con sensores y actuadores.

472. Monitor CRT

Pantalla de tubo de rayos catódicos, una tecnología de visualización antigua y voluminosa.

473. Monitor LCD (Liquid Crystal Display)

Pantalla que usa cristal líquido y es retroiluminada, ligera y eficiente.

474. Monitor LED

Pantalla LCD que usa diodos LED para la retroiluminación, ofreciendo mejor contraste y calidad de imagen.

475. Monitor OLED (Organic Light-Emitting Diode)

Pantalla donde cada píxel emite su propia luz, lo que resulta en negros perfectos y un contraste excepcional.

476. Proyector

Un dispositivo que muestra imágenes sobre una superficie externa grande (pantalla o pared).

477. Teclado

El dispositivo de entrada principal para introducir texto y comandos.

478. Teclado mecánico

Utiliza interruptores físicos individuales en cada tecla, ofreciendo una respuesta táctil y auditiva más precisa.

479. Teclado de membrana

Utiliza capas flexibles de silicona para detectar las pulsaciones, siendo más económico y silencioso.

480. Mouse (Ratón)

El dispositivo apuntador estándar.

481. Mouse óptico

Utiliza un sensor óptico para rastrear el movimiento.

482. Mouse láser

Más preciso que el óptico, funciona en casi cualquier superficie.

483. Touchpad

Un panel táctil integrado en los portátiles para sustituir al ratón.

484. Trackball

Un dispositivo apuntador que utiliza una bola giratoria que el usuario manipula con los dedos o la palma.

485. Joystick

Un control analógico con una palanca central, utilizado principalmente en videojuegos y simuladores de vuelo.

486. Gamepad

Un control de mano con botones y palancas, el estándar en consolas y PC para juegos.

487. Volante de simulación

Un control con forma de volante y pedales para juegos de conducción y carreras.

488. Auriculares

Un dispositivo de salida para audio personal.

489. Micrófono

Un dispositivo de entrada para capturar audio (voz, música, sonido).

490. Webcam

Una cámara utilizada para videollamadas y transmisión de video.

491. Cámara térmica

Un dispositivo especializado que captura imágenes basándose en las diferencias de calor (radiación infrarroja).

492. Impresora de inyección de tinta

Utiliza cartuchos de tinta líquida que son expulsados por boquillas para formar la imagen.

493. Impresora láser

Utiliza **tóner** en polvo y un tambor fotosensible para transferir la imagen al papel, siendo rápida y eficiente.

494. Impresora 3D

Una máquina que construye objetos físicos añadiendo capas sucesivas de material.

495. Plotter

Una impresora de gran formato y precisión especializada en la impresión de planos, mapas o carteles.

496. Memoria USB

Un pequeño dispositivo de almacenamiento flash portátil.

497. Tarjeta SD / MicroSD

Memorias flash portátiles usadas en cámaras, smartphones y tabletas. La MicroSD es la versión más pequeña.

498. Unidad de CD-ROM

Un lector de discos compactos de solo lectura.

499. Unidad de DVD

Un lector y grabador de discos DVD, que ofrecen mayor capacidad que los CD.

500. Unidad Blu-ray

Soporta discos de alta capacidad para video de alta definición (HD).

501. Controlador USB

El chip dentro de la computadora que gestiona la comunicación con los dispositivos USB.

502. XSS (Cross-Site Scripting)

Un ataque que inyecta código malicioso en un sitio web legítimo para robar información de los usuarios.

503. CSRF (Cross-Site Request Forgery)

Este ataque engaña a los usuarios ya autenticados para que realicen acciones no deseadas en un sitio web, como transferir dinero sin su conocimiento.

504. DoS (Denial of Service)

Un ataque que sobrecarga un servidor con tráfico, lo que impide que los usuarios legítimos puedan acceder al servicio.

505. DDoS (Distributed Denial of Service)

Una versión más potente de DoS, que usa una red de computadoras comprometidas, llamadas **botnets**, para lanzar el ataque.

506. Botnet

Una red de dispositivos infectados (**bots**) que los atacantes controlan de forma remota para ejecutar acciones maliciosas a gran escala.

507. Malware

Un término general para cualquier software malicioso que busca dañar, robar o espiar sistemas.

508. Spyware

Un tipo de malware diseñado para recopilar información de un usuario sin su consentimiento.

509. Adware

Un software que muestra publicidad no deseada, a menudo de forma intrusiva.

510. Keylogger

Un malware que registra las pulsaciones de un teclado, lo que permite a los atacantes robar contraseñas y otros datos.

511. Rootkit

Un conjunto de herramientas maliciosas que le dan al atacante un acceso oculto y persistente a un sistema.

512. Troyano

Un malware que se disfraza de un programa legítimo para engañar a los usuarios y que lo instalen, comprometiendo su sistema.

513. Gusano (Worm)

Un tipo de malware que se autorreplica y se propaga automáticamente a través de una red.

514. Virus Informático

Un programa malicioso que se adjunta a archivos legítimos para ejecutarse y propagarse cuando el usuario abre el archivo infectado.

515. Criptografía

La ciencia que usa algoritmos para cifrar y proteger la información. Es la base de la seguridad en las criptomonedas.

516. Criptominería

El uso de recursos de hardware para resolver complejos problemas matemáticos que validan transacciones y generan nuevas criptomonedas.

517. Blockchain

Una tecnología de registro distribuido que almacena datos en bloques encadenados criptográficamente. Su principal característica es que es inmutable y descentralizada.

518. Bitcoin

La primera criptomoneda descentralizada del mundo, creada en blockchain.

519. Ethereum

Una blockchain que, además de su propia criptomoneda, soporta la ejecución de **contratos inteligentes**.

520. Smart Contract (Contrato Inteligente)

Un contrato digital autoejecutable. Una vez programado en una blockchain como Ethereum, se ejecuta automáticamente cuando se cumplen las condiciones predefinidas.

521. NFT (Non-Fungible Token)

Un activo digital único y no intercambiable. Se usa para certificar la propiedad de artículos digitales, como arte o música, en una blockchain.

522. Web 1.0

La primera etapa de la World Wide Web. Era estática y de solo lectura; los usuarios solo consumían contenido sin poder interactuar.

523. Web 2.0

La web de la interacción. Se caracteriza por las redes sociales, los blogs y el contenido generado por los usuarios.

524. Web 3.0

La visión actual de una web descentralizada, donde los datos son propiedad de los usuarios, no de grandes corporaciones. Se basa en tecnologías como **blockchain** y la **inteligencia artificial**.

525. Metaverso

Un entorno virtual persistente e inmersivo donde las personas pueden interactuar social y económicamente a través de avatares digitales.

526. Realidad Virtual (VR)

Una tecnología que crea un entorno digital completamente simulado, aislando al usuario del mundo real.

527. Realidad Aumentada (AR)

Una tecnología que superpone elementos digitales sobre el mundo real, usando dispositivos como el teléfono móvil.

528. Realidad Mixta (MR)

La combinación de AR y VR en un mismo entorno, donde los objetos virtuales pueden interactuar con el mundo real.

529. Gemelo Digital (Digital Twin)

Una representación virtual de un objeto o sistema físico. Permite simular y analizar su comportamiento para optimizar su funcionamiento en el mundo real.

530. Impresión 3D

Una tecnología de fabricación que crea objetos físicos capa por capa a partir de un modelo digital.

531. CAD (Computer-Aided Design)

El uso de software para diseñar y modelar objetos en 2D y 3D.

532. CAM (Computer-Aided Manufacturing)

La tecnología que usa software para controlar y automatizar la maquinaria de fabricación.

533. CNC (Control Numérico Computarizado)

Es una tecnología que utiliza programas de computadora para controlar y automatizar máquinas industriales, como fresadoras y tornos.

534. IoT Industrial (IIoT)

La aplicación del **Internet de las Cosas** en entornos industriales. Utiliza sensores y dispositivos inteligentes para optimizar la producción, mejorar la seguridad y predecir fallos en maquinaria.

535. Smart City

Una ciudad que usa tecnología de la información y la comunicación (**TIC**) para mejorar la eficiencia de los servicios, la calidad de vida y la sostenibilidad.

536. 3G

La tercera generación de redes móviles, que permitió la navegación web y la transferencia de datos básicos en teléfonos móviles.

537. 4G (LTE)

La cuarta generación, que mejoró significativamente la velocidad de internet, haciendo posible el streaming de video y otras aplicaciones de gran ancho de banda.

538. 5G

La quinta generación de redes móviles. Ofrece velocidades ultrarrápidas, muy baja **latencia** y la capacidad de conectar una gran cantidad de dispositivos, lo que es crucial para el IoT.

539. Wi-Fi

Una tecnología inalámbrica que permite que dispositivos se conecten a una red local sin cables.

540. Bluetooth

Un estándar de comunicación inalámbrica de corto alcance, usado para conectar dispositivos como audífonos, teclados y altavoces.

541. NFC (Near Field Communication)

Una tecnología inalámbrica de corto alcance que se usa para pagos móviles y transferencia de datos.

542. RFID (Radio Frequency Identification)

Una tecnología que usa radiofrecuencia para identificar objetos mediante etiquetas electrónicas, común en logística e inventarios.

543. Zigbee

Un protocolo inalámbrico de bajo consumo energético, ideal para dispositivos **IoT** y la automatización del hogar.

544. LoRaWAN

Un protocolo inalámbrico de bajo consumo, pero con un largo alcance, que lo hace ideal para la comunicación entre dispositivos **IoT** en áreas extensas.

545. Satélite Geoestacionario

Un satélite que orbita a la misma velocidad que la rotación de la Tierra, lo que lo mantiene en una posición fija. Se usa para telecomunicaciones y televisión.

546. Starlink

Un proyecto de internet satelital de SpaceX que utiliza una constelación de satélites en **órbita baja** para ofrecer internet global de alta velocidad y baja latencia.

547. Red Mesh

Una topología de red en la que cada dispositivo se conecta con varios otros, lo que la hace más resistente a fallos.

548. Topología de Red

La forma en que se organizan y conectan los dispositivos en una red.

549. Topología Estrella

Una configuración en la que todos los dispositivos se conectan a un punto central, como un router.

550. Topología Bus

Todos los dispositivos comparten un mismo canal de comunicación.

551. Topología Anillo

Una red donde los nodos se conectan formando un círculo. Los datos viajan en una sola dirección.

552. Topología Mixta

Una combinación de varias topologías de red (como estrella, bus o anillo) en un mismo sistema para aprovechar sus ventajas.

553. Modelo OSI (Open Systems Interconnection)

Un modelo de referencia de siete capas que describe cómo los datos se transmiten a través de una red, desde la capa física hasta la de aplicación.

554. Modelo TCP/IP

Un modelo de referencia de cuatro capas (Acceso a la Red, Internet, Transporte y Aplicación) que se utiliza como base para Internet.

555. Capa Física

La primera capa del modelo OSI, responsable de la transmisión de **bits** a través de medios físicos (cables, ondas de radio).

556. Capa de Enlace de Datos

La segunda capa del modelo OSI, que maneja la transmisión de datos entre **nodos vecinos** en una red local y realiza el control de errores.

557. Capa de Red

La tercera capa del modelo OSI, que gestiona el **enrutamiento** de paquetes de datos a través de múltiples redes, utilizando direcciones IP.

558. Capa de Transporte

La cuarta capa del modelo OSI, que asegura la entrega de datos de **extremo a extremo** con protocolos como **TCP** (orientado a conexión) o **UDP** (sin conexión).

559. Realidad Extendida (XR)

Un término paraguas que abarca todas las tecnologías inmersivas, incluyendo la realidad virtual (**VR**), la realidad aumentada (**AR**) y la realidad mixta (**MR**).

560. Edge Computing

Un modelo de procesamiento de datos que lleva los cálculos y el almacenamiento más cerca de donde se originan (el **borde** de la red), reduciendo la latencia.

561. Serverless (Sin Servidor)

Una arquitectura en la que los desarrolladores se enfocan solo en el código de la aplicación (funciones), mientras el proveedor en la nube se encarga de la gestión de los servidores y el escalado.

562. Cloud Native

El desarrollo de aplicaciones que están diseñadas para funcionar de manera óptima en entornos de nube, con principios clave de **escalabilidad**, **resiliencia** y el uso de contenedores y microservicios.

563. Contenedor Docker

Una unidad ligera y portátil que encapsula una aplicación y todas sus dependencias. Garantiza que el software se ejecute igual en cualquier entorno.

564. Kubernetes (K8s)

Una plataforma de **orquestación de contenedores** de código abierto que automatiza el despliegue, el escalado y la gestión de aplicaciones en contenedores.

565. Helm

Un gestor de paquetes para Kubernetes que simplifica la instalación y actualización de aplicaciones preconfiguradas mediante **charts**.

566. Microkernel

Una arquitectura de sistemas operativos que ejecuta solo los servicios esenciales (manejo de interrupciones, gestión de procesos) en el núcleo, delegando el resto en procesos externos para mejorar la estabilidad.

567. Monolito

Una arquitectura de software en la que toda la aplicación se construye como una sola unidad indivisible. Suelen ser difíciles de escalar y actualizar.

568. Arquitectura en capas

Un modelo que organiza el software en niveles (capas), como presentación, lógica de negocio y datos, donde cada capa tiene una función específica y solo interactúa con la capa inmediatamente superior e inferior.

569. Arquitectura hexagonal (Ports and Adapters)

Un modelo de diseño de software que aísla la lógica de negocio (**el núcleo**) de la infraestructura (bases de datos, UI, APIs externas) mediante el uso de "puertos" y "adaptadores".

570. Event Sourcing

Una técnica de persistencia de datos donde el estado de un sistema no se almacena directamente, sino que se reconstruye a partir de una secuencia inmutable de **eventos** registrados.

571. CQRS (Command Query Responsibility Segregation)

Un patrón de diseño de software que separa las operaciones de **lectura** (**Query**) y **escritura** (**Command**) en un sistema, permitiendo optimizar cada lado de forma independiente.

572. API REST (Representational State Transfer)

Un tipo de interfaz que usa métodos HTTP (**GET**, **POST**, etc.) para la comunicación entre aplicaciones. Es simple, popular y fácil de implementar, usando recursos identificables por URL.

573. API GraphQL

Un lenguaje de consulta para APIs que permite a los clientes solicitar **exactamente los datos que necesitan** (y nada más), lo que reduce la sobrecarga de datos y las peticiones múltiples.

574. API SOAP (Simple Object Access Protocol)

Un protocolo de mensajería basado en **XML**, más estructurado y rígido que REST, que se utiliza en entornos empresariales que requieren una alta seguridad y transacciones complejas.

575. Token JWT (JSON Web Token)

Un formato de token seguro y compacto que se utiliza para verificar la identidad de un usuario. Contiene un encabezado, una carga útil (**payload**) y una **firma digital** para garantizar que no ha sido alterado.

576. OAuth 2.0

Un protocolo de autorización que permite a las aplicaciones acceder a los recursos de un usuario (ej. su perfil de Facebook) sin que este tenga que compartir sus credenciales (nombre de usuario y contraseña).

577. OpenID Connect (OIDC)

Es una capa de **autenticación** que se construye sobre OAuth 2.0 para verificar la identidad del usuario de una manera estandarizada y obtener información básica del perfil.

578. Zero Trust Security

Un modelo de seguridad que opera bajo el principio de "nunca confiar, siempre verificar". No confía en nadie, incluso dentro de la red corporativa, por lo que todas las solicitudes de acceso deben ser verificadas.

579. Phishing

Una técnica fraudulenta que engaña a las personas haciéndose pasar por una entidad legítima (banco, empresa) a través de correo electrónico o sitios web para robar datos o credenciales.

580. Vishing

La variante de **phishing** que utiliza **llamadas telefónicas** falsas, a menudo con sistemas de voz automatizados, para manipular a las personas y obtener información sensible.

581. Smishing

Una estafa que utiliza **mensajes SMS** con enlaces maliciosos o solicitudes urgentes de datos, buscando que la víctima actúe por impulso.

582. Malware

Un término general para el **software malicioso** diseñado para dañar, espiar o robar información de un sistema.

583. Ransomware

Un tipo de malware que **cifra los archivos** de un sistema y exige un pago, generalmente en criptomonedas, a cambio de la clave para restaurar el acceso.

584. Adware

Un programa que muestra anuncios no deseados y, a menudo, no solicitados, y puede afectar el rendimiento y la privacidad de un dispositivo.

585. Spyware

Un software que se instala sin permiso del usuario para espiar, monitorear la actividad y recopilar información personal o sensible.

586. Keylogger

Un programa que registra cada pulsación de tecla para robar contraseñas, números de tarjetas de crédito y otros datos sensibles.

587. Botnet

Una red de dispositivos infectados (**bots**) que son controlados de forma remota por un atacante (**botmaster**) para lanzar ataques masivos, como el DDoS.

588. DDoS (Distributed Denial of Service)

Un ataque de **denegación de servicio distribuido** que usa una **botnet** (múltiples fuentes) para saturar un servidor con tráfico, haciéndolo inaccesible para los usuarios legítimos.

589. Honeypot (Tarro de Miel)

Un sistema trampa o recurso de red que se diseña para **atraer** y **analizar atacantes**, simulando una vulnerabilidad o un sistema valioso para estudiar sus técnicas sin poner en riesgo los sistemas reales.

590. IDS (Sistema de Detección de Intrusiones)

Un sistema que monitorea el tráfico de red para **detectar** actividad sospechosa, violaciones de políticas o ataques. Solo alerta, no actúa.

591. IPS (Sistema de Prevención de Intrusiones)

A diferencia de un IDS, un IPS no solo detecta amenazas, sino que también las **bloquea activamente** o toma medidas correctivas en tiempo real, como cortar la conexión.

592. Antivirus

Un programa de seguridad básico que protege principalmente contra **virus informáticos** conocidos, detectándolos y eliminándolos.

593. Antimalware

Un software de seguridad más completo que protege contra una gama más amplia de amenazas, incluyendo **ransomware**, **spyware**, **adware** y **troyanos**, además de los virus.

594. Backup incremental

Una copia de seguridad que guarda solo los archivos que se han **modificado** desde el **último** respaldo (ya sea incremental o completo), siendo el más rápido de crear pero el más lento de restaurar.

595. Backup diferencial

Una copia de seguridad que guarda todos los cambios realizados desde el **último backup completo**. Tarda más que el incremental, pero es más rápido de restaurar.

596. Backup completo

Una copia de seguridad total de todos los datos en un sistema. Es el más lento de crear, pero el más rápido y sencillo de restaurar.

597. RAID 0 (Striping)

Una configuración de discos que mejora la **velocidad** de lectura/escritura al distribuir los datos entre varias unidades, pero **no ofrece redundancia**; si un disco falla, se pierden todos los datos.

598. RAID 1 (Mirroring)

Duplica los datos exactamente en dos discos, creando un "**espejo**" para una alta **seguridad** y redundancia. Ofrece tolerancia a fallos, pero solo usa el 50% de la capacidad total.

599. RAID 5

Distribuye datos y la información de **paridad** (corrección de errores) entre varios discos (mínimo 3), logrando un buen equilibrio entre rendimiento, seguridad y aprovechamiento del espacio.

600. RAID 10 (1+0)

Combina **RAID 1** (espejo) y **RAID 0** (segmentación) para ofrecer lo mejor de ambos mundos: **alta velocidad** (por RAID 0) y **alta redundancia** (por RAID 1).

601. NAS (Network Attached Storage)

Un dispositivo de almacenamiento conectado directamente a una red que permite el acceso a archivos desde múltiples dispositivos (computadoras, teléfonos) de forma centralizada. Es ideal para hogares y pequeñas empresas.

602. SAN (Storage Area Network)

Una red dedicada y de alta velocidad que proporciona almacenamiento de **alto rendimiento** a nivel de bloque para servidores. Se usa en grandes empresas para bases de datos y aplicaciones críticas.

603. VPS (Servidor Virtual Privado)

Un servidor virtual que ofrece **recursos dedicados** (RAM, CPU) a un cliente, pero que comparte el hardware físico con otros VPS. Es un equilibrio entre el hosting compartido y el dedicado.

604. Hosting compartido

Un servicio donde varios sitios web comparten los mismos recursos de un único servidor físico. Es la opción más económica, pero el rendimiento de un sitio puede verse afectado por los otros.

605. Hosting dedicado

Un servidor completo y **exclusivo** para un solo cliente, lo que le da un control total sobre los recursos, la configuración y el software. Ideal para sitios de alto tráfico o aplicaciones complejas.

606. CDN (Content Delivery Network)

Una red de servidores distribuidos geográficamente que reduce la **latencia** al entregar contenido (imágenes, videos, archivos estáticos) a los usuarios desde el servidor más cercano a ellos.

607. Proxy

Un servidor intermediario que canaliza las solicitudes entre un cliente y un servidor de destino. Se usa para mejorar la **seguridad**, el **anonimato** o para aplicar políticas de acceso (filtrado de contenido).

608. Tor (The Onion Router)

Una red que enruta las comunicaciones a través de múltiples nodos cifrados (capas, como una "cebolla") para anonimizar el tráfico y la ubicación del usuario.

609. Deep Web

La parte de Internet no indexada por los motores de búsqueda (ej. Google), que incluye bases de datos privadas, correos electrónicos o contenido protegido por contraseña. Es la mayor parte de internet.

610. Dark Web

Un subconjunto de la **Deep Web** que requiere herramientas especiales (como Tor) para acceder. Se asocia a actividades ilícitas y mercados negros, debido a su alto nivel de anonimato.

611. Surface Web

La parte de Internet que se puede encontrar y acceder a través de buscadores convencionales (Google, Bing). Es solo la punta del iceberg de toda la información en línea.

612. Indexación

El proceso de catalogar, registrar y organizar páginas web en una base de datos de un motor de búsqueda, haciéndolas accesibles para las consultas de los usuarios.

613. Rastreador web (Web Crawler / Spider)

Un programa automatizado que navega metódicamente por la web siguiendo enlaces y recopilando información de las páginas para la **indexación** del motor de búsqueda.

614. CTR (Click-Through Rate)

El porcentaje de veces que un usuario hace clic en un enlace (ya sea de búsqueda o publicitario) en relación con el número de veces que fue visto (**impresiones**). Mide la efectividad del título y la descripción.

615. KPI (Indicador Clave de Rendimiento)

Una métrica cuantificable que mide el desempeño de una estrategia, un proceso o un objetivo específico. Ejemplos: Ventas mensuales, tasa de abandono de carrito, etc.

616. ROI (Retorno de la Inversión)

Una métrica financiera que evalúa la rentabilidad de una inversión, comparando el beneficio obtenido con el costo de la inversión. Se expresa generalmente en porcentaje.

617. Growth Hacking

Una estrategia de marketing creativa y experimental enfocada en el **crecimiento rápido** de usuarios o ingresos, a menudo con bajos recursos, utilizando métodos poco convencionales.

618. Marketing Automation

El uso de software para automatizar tareas de marketing repetitivas, como el envío de correos electrónicos segmentados, la gestión de leads o la publicación en redes sociales, para mejorar la eficiencia.

619. CRM (Customer Relationship Management)

Un sistema que gestiona y rastrea todas las interacciones con los clientes y prospectos. Ayuda a organizar la información de ventas, marketing y servicio al cliente para mejorar las relaciones comerciales.

620. ERP (Enterprise Resource Planning)

Un software integral que unifica y automatiza los procesos clave de negocio de una empresa, como finanzas, recursos humanos, inventario y producción, en un único sistema.

621. BI (Business Intelligence)

Herramientas y metodologías que transforman datos brutos en **información útil** a través de informes y tableros para la toma de **decisiones estratégicas** basadas en evidencia.

622. Data Cleaning (Limpieza de Datos)

El proceso de encontrar, corregir o eliminar errores, duplicados o inconsistencias en un conjunto de datos para mejorar su calidad y prepararlos para el análisis.

623. Machine Learning (ML)

Un subcampo de la **IA** que permite a las máquinas **aprender** de los datos para identificar patrones y tomar decisiones con una mínima o nula programación explícita.

624. Deep Learning

Un campo especializado del **ML** que utiliza **redes neuronales profundas** con múltiples capas ocultas para aprender patrones complejos directamente de grandes cantidades de datos.

625. Reinforcement Learning (Aprendizaje por Refuerzo)

Un tipo de **ML** donde un agente aprende a tomar decisiones en un entorno (ej. un juego) a través de un sistema de prueba y error, guiado por un sistema de **recompensas y castigos**.

626. Red neuronal convolucional (CNN)

Un tipo de red neuronal utilizada principalmente para el **reconocimiento y procesamiento de imágenes** (visión por computadora), ya que puede identificar patrones espaciales y jerárquicos.

627. Red neuronal recurrente (RNN)

Una red neuronal diseñada para procesar datos **secuenciales**, como texto, voz y series de tiempo, ya que sus conexiones permiten que la información persista (tengan "memoria").

628. Text-to-Speech & Speech-to-Text

**Text-to-Speech** (TTS) convierte texto escrito en voz. **Speech-to-Text** (STT) transcribe la voz a texto (reconocimiento de voz).

629. OCR (Reconocimiento Óptico de Caracteres)

Una tecnología que reconoce y extrae texto de imágenes, fotografías o documentos escaneados, convirtiéndolo en texto editable.

630. Diffusion Models

Algoritmos de **IA generativa** que crean imágenes (o datos) a partir de ruido aleatorio, refinándolo gradualmente en una secuencia de pasos hasta crear una salida coherente. Son la base de herramientas modernas de arte IA.

631. GAN (Generative Adversarial Network)

Un par de redes neuronales (un **generador** que crea contenido y un **discriminador** que lo juzga) que compiten entre sí para producir resultados extremadamente realistas.

632. Dataset (Conjunto de Datos)

Una colección estructurada de datos (filas y columnas) utilizada para entrenar y probar modelos de **Machine Learning**.

633. Feature Engineering

El proceso de seleccionar, transformar y crear nuevas variables (**features**) a partir de datos brutos para mejorar significativamente el rendimiento de un modelo de **ML**.

634. Overfitting

**Overfitting** ocurre cuando un modelo aprende los datos de entrenamiento tan bien, incluyendo el ruido y los detalles irrelevantes, que **no logra generalizar** a datos nuevos. *El error en la descripción original fue que la 634 describe Underfitting*.

635. Underfitting

**Underfitting** sucede cuando un modelo es demasiado simple y no puede captar la complejidad de los datos ni los patrones relevantes, lo que resulta en un mal rendimiento tanto en datos de entrenamiento como en nuevos datos. *El error en la descripción original fue que la 635 describe Overfitting*.

636. Validación Cruzada (Cross-Validation)

Un método estadístico que divide un conjunto de datos en varios subconjuntos (folds) para entrenar y evaluar el rendimiento de un modelo de manera más robusta y evitar el **overfitting**.

637. Accuracy (Exactitud)

El porcentaje de **predicciones correctas** (tanto positivos como negativos) sobre el total de predicciones realizadas. Es la métrica de rendimiento más simple.

638. Precision (Precisión)

La proporción de predicciones **positivas** que fueron realmente correctas (**Verdaderos Positivos** / (Verdaderos Positivos + Falsos Positivos)). Indica qué tan confiable es la etiqueta "positivo" del modelo.

639. Recall (Sensibilidad/Exhaustividad)

La proporción de **casos positivos reales** que el modelo logró identificar correctamente (**Verdaderos Positivos** / (Verdaderos Positivos + Falsos Negativos)). Mide la capacidad del modelo para encontrar todos los positivos reales.

640. F1-Score

El valor promedio armónico entre la **Precisión** y el **Recall**. Proporciona una única métrica que equilibra ambos valores, siendo útil en datasets desbalanceados.

641. Curva ROC (Receiver Operating Characteristic)

Un gráfico que muestra la relación entre la **Tasa de Verdaderos Positivos** (Recall) y la **Tasa de Falsos Positivos** en diferentes umbrales de clasificación.

642. AUC (Área Bajo la Curva)

Una métrica que representa el área total bajo la **Curva ROC**. Mide la capacidad de un modelo para distinguir entre clases. Un valor de **1.0** es perfecto, **0.5** es aleatorio.

643. Confusion Matrix (Matriz de Confusión)

Una tabla que resume el rendimiento de un modelo de clasificación mostrando el número de aciertos (**Verdaderos Positivos/Negativos**) y errores (**Falsos Positivos/Negativos**) para cada clase.

644. Batch

Un pequeño subconjunto de datos de entrenamiento que se procesa en una sola iteración (paso de actualización) del algoritmo de aprendizaje.

645. Gradient Descent (Descenso de Gradiente)

Un algoritmo de optimización que ajusta los parámetros (pesos) de un modelo de **ML** de forma iterativa para **minimizar su error** o función de pérdida.

646. Learning Rate (Tasa de Aprendizaje)

Un **hiperparámetro** que controla cuánto se ajustan los parámetros del modelo en cada iteración del **Descenso de Gradiente**. Un valor alto puede hacer que el modelo "salte" la solución; uno muy bajo lo hace lento.

647. Regularization (Regularización)

Una técnica utilizada para evitar el **overfitting** al penalizar los modelos que tienen coeficientes o pesos demasiado grandes, forzándolos a ser más simples.

648. Dropout

Un método en redes neuronales que desactiva aleatoriamente algunas neuronas (las pone a cero) durante cada paso de entrenamiento para evitar la **overfitting** y mejorar la robustez.

649. Normalization (Normalización)

Escalar los datos a un rango común (por ejemplo, de **0 a 1**) para mejorar la estabilidad del entrenamiento y asegurar que todas las características tengan el mismo peso.

650. Standardization (Estandarización)

Transformar los datos para que tengan una **media de 0** y una **desviación estándar de 1** (distribución normal). Es útil cuando el modelo asume datos con distribución Gaussiana.

651. One-hot encoding (Codificación One-Hot)

Un método para convertir variables **categóricas** (como nombres de ciudades) en **vectores binarios dispersos** (compuestos por ceros y un solo uno) para que puedan ser procesadas por modelos de Machine Learning.

652. Embeddings

Representaciones **vectoriales densas** (listas de números decimales) de palabras u objetos que capturan su **significado semántico** y sus relaciones contextuales. Palabras con significados similares están cerca en el espacio vectorial.

653. Word2Vec

Un modelo de aprendizaje no supervisado que crea **embeddings** para palabras basándose en su **contexto** dentro de un gran corpus de texto. Famoso por permitir operaciones como "Rey - Hombre + Mujer = Reina".

654. GloVe (Global Vectors for Word Representation)

Un algoritmo que crea **embeddings** basándose en las **estadísticas de co-ocurrencia** de palabras (con qué frecuencia aparecen juntas en el texto) en un corpus.

655. BERT (Bidirectional Encoder Representations from Transformers)

Un poderoso modelo de lenguaje desarrollado por Google basado en la arquitectura **Transformer**. Es excelente para entender el **contexto bidireccional** de las palabras en una oración, prestando atención a lo que viene antes y después.

656. LSTM (Long Short-Term Memory)

Un tipo de **Red Neuronal Recurrente** que es particularmente bueno para **recordar dependencias a largo plazo** en datos secuenciales (como texto), resolviendo el problema del desvanecimiento del gradiente de las RNN tradicionales.

657. Línea

Elemento básico que une dos puntos. Puede ser recta, curva, gruesa, delgada, y se utiliza para dirigir la vista, crear contornos o separar espacios.

658. Forma

Área cerrada o figura bidimensional con contornos definidos, como un círculo, un cuadrado o una figura orgánica. Son la base de los objetos visuales.

659. Color

Elemento de diseño que proviene de la luz reflejada o emitida. Es clave para crear ambiente, jerarquía, contraste y transmitir emociones.

660. Textura

Cualidad superficial de una forma, que puede ser **táctil** (si se imprime) o **visual** (ilusión de rugosidad, suavidad, etc.).

661. Espacio

El área alrededor, dentro o entre los elementos de diseño (también conocido como **espacio negativo** o blanco). Es vital para la composición, la legibilidad y para crear contraste.

662. Equilibrio

Distribución uniforme del **peso visual** en un diseño. Puede ser **simétrico** (elementos idénticos a ambos lados de un eje central) o **asimétrico** (elementos diferentes que se compensan en peso).

663. Contraste

Diferencia notable entre dos o más elementos, como el color (claro/oscuro), la forma (grande/pequeño), o la tipografía (fina/negrita), utilizado para hacerlos destacar y crear jerarquía.

664. Repetición

Uso consistente de elementos (color, forma, tipografía) en un diseño para crear unidad, cohesión y reforzar una idea.

665. Alineación

La disposición de los elementos en relación con un eje o borde común para crear un diseño ordenado, estructurado y profesional.

666. Proximidad

Principio de diseño que agrupa elementos relacionados espacialmente para formar una unidad visual, lo que mejora la organización y la comprensión.

667. Jerarquía visual

Organización de los elementos de diseño (a través del tamaño, color, contraste) para guiar la atención del espectador en un orden de importancia, asegurando que el mensaje clave sea visto primero.

668. Tipografía

El arte y la técnica de organizar el texto para hacerlo legible, atractivo y significativo, incluyendo la elección de la fuente, el tamaño y el espaciado.

669. Composición

Disposición general y organización intencional de todos los elementos visuales (líneas, formas, colores, texto) en una obra de diseño.

670. Maquetación

Proceso de organizar el texto y las imágenes de manera estructurada en una página o pantalla, especialmente en publicaciones como libros, revistas o sitios web.

671. Retícula (Grid)

Guía estructural invisible (hecha de líneas y columnas) que ayuda a organizar el contenido de forma coherente, manteniendo la **alineación** y el orden en el diseño.

672. Adobe Photoshop

Editor de imágenes de **mapa de bits** estándar de la industria, utilizado para retoque fotográfico, manipulación de imágenes y arte digital.

673. Adobe Illustrator

Software de diseño **vectorial** para crear logotipos, iconos, ilustraciones, fuentes y otros gráficos que se pueden escalar indefinidamente sin perder calidad.

674. Adobe InDesign

Software de **maquetación** profesional utilizado para crear publicaciones de varias páginas, como libros, revistas, folletos y catálogos.

675. CorelDRAW

Editor de **gráficos vectoriales** que compite directamente con Adobe Illustrator, popular en la industria de la impresión y la señalización.

676. GIMP (GNU Image Manipulation Program)

Editor de gráficos de **código abierto** y gratuito, similar a Photoshop, que maneja imágenes de **mapa de bits**.

677. Sketch

Una herramienta de diseño **vectorial** popular entre diseñadores web y de **UI/UX** (interfaz de usuario/experiencia de usuario), especialmente en el ecosistema macOS.

678. Affinity Designer

Una alternativa asequible a Illustrator que ofrece herramientas **vectoriales** y de **mapa de bits** combinadas en una sola aplicación.

679. Adobe Fresco

Aplicación de dibujo y pintura digital con pinceles **vectoriales** y de **mapa de bits** (píxeles), diseñada especialmente para dispositivos táctiles.

680. Adobe Express

Plataforma de diseño gráfico todo en uno, enfocada en la creación **rápida y sencilla** de contenido para redes sociales y marketing, accesible en línea.

681. Capas (Layers)

Elementos apilados en un diseño que permiten editar y manipular objetos, imágenes o efectos de forma **independiente** sin afectar el resto del contenido.

682. Máscara de capa (Layer Mask)

Una técnica que utiliza valores de gris, negro o blanco para **ocultar o revelar** partes de una capa sin borrarlas permanentemente, permitiendo la edición no destructiva.

683. Pincel (Brush)

Herramienta que simula el trazo de un pincel o lápiz, con diversos tamaños, formas y estilos. Fundamental en el trabajo de mapa de bits (Photoshop, GIMP).

684. Pluma (Pen Tool)

Herramienta de software vectorial (Illustrator) para crear líneas y curvas precisas utilizando **puntos de ancla** y **manejadores** (puntos de control).

685. Varita mágica (Magic Wand)

Herramienta de selección automática que selecciona áreas de una imagen basándose en la **similitud de color** o el brillo de los píxeles adyacentes.

686. Recorte (Crop)

Herramienta para eliminar las partes no deseadas de una imagen, ajustando sus límites para mejorar la composición o el enfoque.

687. Escala

Función para aumentar o disminuir el tamaño de un objeto, una forma o una imagen, a menudo de forma proporcional para mantener la integridad.

688. Modos de fusión (Blending Modes)

Opciones que controlan cómo los píxeles de una capa interactúan o se combinan con los píxeles de las capas inferiores, creando efectos creativos de luz, color o textura.

689. Trazo (Stroke)

Contorno o borde que se aplica alrededor de una forma, objeto o texto. En software vectorial, es una propiedad separada del relleno.

690. Kerning

Ajuste del **espacio horizontal** entre **dos caracteres específicos** en un texto para mejorar el equilibrio visual de las letras.

691. Tracking

Ajuste uniforme del **espacio horizontal** entre **todos los caracteres** en un bloque de texto o una palabra completa.

692. Interlineado (Leading)

Espacio **vertical** entre las líneas base de un texto. Es crucial para la legibilidad.

693. Vectorización (Image Trace)

Conversión de una imagen de **mapa de bits** (píxeles) en un **gráfico vectorial** (fórmulas matemáticas) para que pueda ser escalada sin pérdida de calidad.

694. Rasterización

Conversión de un **gráfico vectorial** en una imagen de **mapa de bits** (píxeles) a una resolución específica, lo que la hace susceptible a la pixelación si se agranda.

695. Trazado de recorte (Clipping Mask)

Uso de una forma o capa (la máscara) para definir la **visibilidad** de las capas inferiores. Solo se ven las partes de la capa inferior que están dentro de la forma de la máscara.

696. Filtros

Efectos preestablecidos que se aplican a imágenes para cambiar rápidamente su apariencia, como desenfoque, nitidez, o alteraciones de color y textura.

697. Degradado (Gradient)

Transición suave y gradual de un color a otro, de un color a transparencia, o de un tono claro a uno oscuro.

698. Mesa de trabajo (Artboard)

El área de dibujo o lienzo donde se crea el diseño, especialmente en software vectorial (Illustrator). Permite tener múltiples diseños o páginas en un solo archivo.

699. Pantone

Un sistema estandarizado de igualación de colores (PMS) utilizado en todo el mundo, esencial para la **impresión** para garantizar que los colores sean consistentes, independientemente del equipo que se utilice.

700. Cuentagotas (Eyedropper)

Herramienta utilizada para **tomar muestras de color** de cualquier punto en el diseño o la imagen para establecerlo como el color de primer plano o el color de relleno.

701. Gráficos de mapa de bits (Bitmap / Ráster)

Imágenes formadas por una cuadrícula de **píxeles** de diferentes colores (como fotografías o archivos JPG, PNG). Su resolución es fija.

702. Gráficos vectoriales

Imágenes construidas a partir de **fórmulas matemáticas** que definen puntos, líneas y curvas. Son escalables a cualquier tamaño **sin pérdida de calidad** (como logotipos o archivos SVG, AI).

703. Píxel

La **unidad más pequeña** de una imagen digital de **mapa de bits**. Cada píxel almacena información de un solo color.

704. Punto por pulgada (DPI - Dots Per Inch)

Medida de la **resolución de una imagen** para **impresión**. Cuanto mayor sea el DPI, mayor será el detalle de la imagen impresa.

705. Píxeles por pulgada (PPI - Pixels Per Inch)

Medida de la **resolución de una imagen en pantalla** (monitores, móviles). Indica la densidad de píxeles en una pulgada cuadrada. Es crucial para el diseño web y UI/UX.

706. JPG/JPEG (Joint Photographic Experts Group)

Formato de compresión **con pérdida**, ideal para **fotografías** y imágenes complejas en web, ya que logra tamaños de archivo muy pequeños a costa de sacrificar algo de calidad.

707. PNG (Portable Network Graphics)

Formato de imagen con **compresión sin pérdida**, que soporta **transparencia** de alta calidad. Ideal para logotipos, iconos e ilustraciones en el diseño digital.

708. GIF (Graphics Interchange Format)

Formato de imagen que permite **animaciones simples** y soporta solo hasta **256 colores**. Es útil para gráficos sencillos y clips de video cortos sin audio.

709. TIFF (Tagged Image File Format)

Formato de imagen de **alta calidad y sin pérdida**, utilizado en impresión profesional, escaneo de alta resolución y archivos maestros, ya que puede guardar información de color avanzada.

710. RAW

Formato de archivo de imagen **sin procesar** de una cámara digital. Conserva toda la información capturada por el sensor sin aplicar compresión ni ajustes de color, dando máximo control en la edición.

711. PSD (Photoshop Document)

Formato de archivo **nativo de Adobe Photoshop**. Conserva todas las **capas**, máscaras, textos y la información editable del diseño de mapa de bits.

712. AI (Adobe Illustrator Artwork)

Formato de archivo **nativo de Adobe Illustrator**. Mantiene la información **vectorial** completa, permitiendo la edición de puntos, curvas y texto escalable.

713. EPS (Encapsulated PostScript)

Formato de archivo **vectorial** antiguo, pero compatible con múltiples aplicaciones. Se utiliza a menudo para la transferencia de logotipos y gráficos vectoriales a programas de maquetación.

714. SVG (Scalable Vector Graphics)

Formato de imagen **vectorial** basado en XML. Es esencialmente un código que dibuja el gráfico, lo que lo hace perfecto para gráficos web (iconos, ilustraciones) que deben escalar sin pixelación.

715. RGB (Rojo, Verde, Azul)

Modelo de color **aditivo** usado para la visualización en **pantallas digitales** (monitores, móviles). Se basa en la mezcla de luz para crear colores. Es el modelo estándar para el diseño de UI/UX y web.

716. CMYK (Cian, Magenta, Amarillo, Negro)

Modelo de color **sustractivo** usado en la **impresión**. Se basa en la mezcla de tintas. Es el modelo estándar para cualquier diseño destinado a ser impreso físicamente.

717. Modo Escala de Grises

Modo de color que utiliza únicamente tonos de gris, desde el negro puro hasta el blanco puro, sin información de color (matiz) adicional.

718. Color plano (Spot Color / Flat Color)

Un color sólido, sin degradados ni sombras. En impresión, se refiere a una tinta premezclada (ej. Pantone). En diseño digital, se refiere a un color **uniforme** aplicado para un estilo limpio (**Flat Design**).

719. Saturación

La **intensidad o pureza** de un color. Un color muy saturado es vibrante; un color poco saturado es más grisáceo o apagado.

720. Brillo (Value / Luminosity)

La claridad u oscuridad relativa de un color, independientemente de su matiz. Controla cuán cerca está un color del blanco o del negro. En UX/UI, se relaciona con el impacto visual y la **legibilidad**.

721. Tinte (Tint)

Un color (matiz) mezclado con **blanco** para hacerlo más claro. En diseño, se usa para crear paletas análogas o variaciones sutiles.

722. Sombra (Shade)

Un color (matiz) mezclado con **negro** para hacerlo más oscuro.

723. Opacidad

La propiedad de un objeto o color de ser más o menos **transparente**. Una opacidad del 100% es completamente sólido; 0% es invisible. Fundamental en UI para crear jerarquía.

724. Paleta de colores

Un conjunto predefinido de colores seleccionados por el diseñador para mantener la **coherencia visual** y la identidad de marca en un proyecto.

725. Fuente (Typeface/Font)

El conjunto de caracteres (letras, números, símbolos) con un estilo, peso y diseño uniformes. Su elección es clave para la **legibilidad** y el tono emocional de la interfaz.

726. Familia tipográfica (Type family)

Conjunto completo de fuentes relacionadas que comparten un diseño base, pero varían en **peso** (delgado, negrita), **estilo** (cursiva, regular) y a veces ancho (condensado).

727. Serif

Fuente con pequeños **trazos o remates decorativos** en los extremos de los caracteres (ej. Times New Roman). Se usan tradicionalmente para texto largo impreso por su legibilidad en papel.

728. Sans-serif

Fuentes **"sin serifa"** (sin remates). Son las más utilizadas en **pantallas digitales** (web, móvil) por su claridad y limpieza a bajas resoluciones (ej. Arial, Helvetica).

729. Cuerpo (Size/Point size)

El tamaño de la fuente, medido típicamente en **puntos (pt)** o **píxeles (px)**. Afecta directamente la legibilidad y la jerarquía visual del texto.

730. Peso (Weight) y Estilo (Style)

**Peso** se refiere al grosor del trazo de la tipografía (ej. Regular, Bold, Light). **Estilo** se refiere a la variación del carácter (ej. Italic, Underline).

731. Caja alta (Uppercase)

Uso de letras **mayúsculas**. Se usan para títulos, acrónimos, o en contextos donde se quiere dar énfasis. El uso excesivo reduce la legibilidad.

732. Caja baja (Lowercase)

Letras **minúsculas**. Son el estándar para el cuerpo de texto por su mayor legibilidad y la familiaridad con la que se perciben las formas de las palabras.

733. Viudas y Huérfanas

Errores de maquetación: Una **viuda** es la última línea de un párrafo que queda sola al principio de una columna o página. Una **huérfana** es la primera línea de un párrafo que queda sola al final de una columna o página.

734. Espaciado de palabra (Word Spacing)

El espacio horizontal entre palabras en una línea de texto. El ajuste es crucial en la justificación para evitar "ríos" tipográficos.

735. Espaciado de letra (Letter Spacing / Tracking)

Ajuste uniforme del espacio horizontal entre todos los caracteres en un bloque de texto. Aumentarlo (abrir) o reducirlo (cerrar) afecta la densidad del texto.

736. Río (River)

Un error tipográfico donde los **espacios en blanco** entre palabras en líneas consecutivas se alinean verticalmente, creando un espacio hueco que fluye a través del párrafo y distrae al lector.

737. Silabación (Hyphenation)

División automática de palabras al final de una línea mediante un guion para justificar el texto y reducir los espacios excesivos.

738. Glifo

La **representación individual** (el dibujo) de un carácter. Por ejemplo, la "a" minúscula y la "A" mayúscula son glifos diferentes del mismo carácter (letra A).

739. Transparencia

Propiedad que permite ver a través de un objeto o una imagen, esencial para superponer elementos en diseño de interfaz (UI).

740. Sombra paralela (Drop Shadow)

Efecto que simula una sombra proyectada detrás de un objeto o texto, dando la ilusión de elevación y profundidad.

741. Bisel y relieve (Bevel and Emboss)

Efecto de capa que da profundidad y tridimensionalidad a una forma o texto, simulando bordes biselados o un material grabado.

742. Desaturar e Invertir

**Desaturar** es reducir la intensidad (pureza) de los colores de una imagen, acercándola al blanco y negro. **Invertir** es intercambiar los colores de una imagen por sus opuestos complementarios (negro se vuelve blanco, etc.).

743. Desenfoque (Blur)

Efecto que suaviza o difumina los detalles de una imagen. Se usa para crear profundidad, suavizar texturas o dirigir la atención.

744. Enfoque (Sharpen)

Efecto que aumenta la nitidez de los bordes y detalles en una imagen, haciendo que parezca más definida.

745. Viñeteado (Vignette)

Oscurecimiento o atenuación progresiva de los bordes o esquinas de una imagen para dirigir la atención al centro.

746. Curvas de color (Curves)

Herramienta avanzada de edición de imágenes que permite ajustar el brillo, el contraste y el tono de una imagen con gran **precisión** a través de una línea diagonal en un gráfico.

747. Niveles (Levels)

Herramienta de edición que utiliza un **histograma** para corregir el tono y el contraste de una imagen, ajustando los puntos blanco, negro y gris medio.

748. Efectos de lente (Lens Effects)

Simulación de fenómenos ópticos que ocurren con una lente de cámara real, como el **destello de lente (lens flare)** o la distorsión.

749. Estilos de capa (Layer Styles)

Conjunto de efectos no destructivos (como sombras, biseles, superposiciones de color) que se aplican a una capa en Photoshop y que se pueden modificar o eliminar en cualquier momento.

750. Trazado de corte (Clipping Path)

Un **trazo vectorial** que se utiliza para definir el límite de una imagen, recortándola limpiamente de su fondo. Es vital para la impresión y la colocación de imágenes en maquetación.

751. Distorsión

Modificación geométrica de una imagen o forma (ej. perspectivas, deformaciones libres) para crear efectos visuales o corregir imperfecciones ópticas.

752. Perspectiva

Representación de un objeto en tres dimensiones sobre una superficie plana.

753. Ilustración

Representación gráfica de un concepto o idea, a menudo creada con software vectorial.

754. Boceto

Dibujo preliminar de una idea, a menudo hecho con la herramienta de pincel.

755. Animación de movimiento (Motion Graphics)

Animación de elementos gráficos para crear movimiento visual.

756. Keyframe

Fotograma clave en una animación que define el inicio y el final de una transición.

757. Interpolación (Tweening)

Proceso de generar los fotogramas intermedios entre dos keyframes.

758. Dibujo con tableta gráfica

Uso de una tableta con un lápiz digital para dibujar directamente en el software.

759. Calado

Delimitación de un área en una imagen para aislarla o modificarla.

760. Pintado digital

Creación de ilustraciones utilizando herramientas de pintura y pinceles digitales.

761. Vectorización automática

Función que convierte una imagen de mapa de bits en un vector de forma automática.

762. Prototipo

Versión funcional de un diseño, utilizada para probar la interacción y la experiencia de usuario.

763. Paleta de herramientas

Conjunto de iconos que representan las herramientas disponibles en el software.

764. Menú contextual

Menú que aparece al hacer clic con el botón derecho en un objeto o área.

765. Atajo de teclado

Combinación de teclas que activa una función del software.

766. Exportar

Guardar un archivo en un formato específico para su uso en otra aplicación o para la web.

767. Importar

En desarrollo de software, "importar de teclado" se refiere a la acción de capturar la entrada de datos que un usuario ingresa a través del teclado.

768. Previsualización

Ver el diseño tal como se verá en el resultado final (web).

769. Flujo de trabajo

Conjunto de pasos que se siguen para completar un proyecto de diseño.

770. Plugins

Extensiones de software que añaden nuevas funcionalidades.

771. Fuentes embebidas

Inclusión de las fuentes en un archivo para que se vean correctamente en cualquier sistema.

772. Gestión de color

Proceso de asegurar que los colores se mantengan consistentes en diferentes dispositivos y medios.

773. Resolución de impresión

Implica gestionar y configurar parámetros para asegurar que los elementos visuales (imágenes, textos, gráficos) se reproduzcan de forma nítida y precisa, en lugar de aparecer pixelados o borrosos.

774. Margen

Espacio entre el borde de la página y el contenido impreso.

775. Sangrado (Bleed)

Área de diseño que se extiende más allá de la línea de corte para evitar bordes blancos después de la impresión.

776. Corte (Trim)

El corte de control es una técnica de programación que permite procesar y generar informes sobre datos que ya están agrupados o jerarquizados por algún criterio.

777. Prensa digital

Para la prensa digital sobre desarrollo de software, existe una amplia variedad de plataformas que ofrecen noticias, tutoriales y análisis de tendencias para desarrolladores.

778. Renderizado

Proceso de generar una imagen a partir de un modelo 3D o una escena animada.

779. Modelado 3D

Creación de modelos tridimensionales utilizando software especializado.

780. Mapas de bits de alta fidelidad

Imágenes con gran profundidad de color y detalles, a menudo en formato RAW o TIFF.

781. Curvas de Bézier

Las curvas de Bézier son representaciones matemáticas de curvas suaves, fundamentales en el desarrollo de software para gráficos por computadora, animación, diseño asistido por computadora (CAD) y tipografía digital.

782. Trazado de píxeles

Conversión de una imagen de mapa de bits en un trazo vectorial manualmente.

783. Recorte de fondo

784. HDR (High Dynamic Range)
